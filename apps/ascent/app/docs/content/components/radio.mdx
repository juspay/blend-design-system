```tsx

}
```

## Features

- **Size Variants**: Two size options (small, medium) for different contexts
- **Group Management**: Automatic mutual exclusivity within RadioGroup containers
- **Rich Content Support**: Optional subtext and slot elements for enhanced context
- **State Management**: Support for controlled and uncontrolled modes
- **Form Integration**: Native HTML radio input with proper form binding
- **Visual States**: Required field indicators, error states, and disabled states
- **Keyboard Navigation**: Full arrow key navigation within groups with auto-selection
- **Individual Controls**: Can be used standalone outside of groups
- **Automatic IDs**: Generated unique IDs when not provided
- **Accessibility First**: ARIA compliance, screen reader support, and proper semantics
- **Custom Styling**: Comprehensive token system for visual customization
- **Focus Management**: Clear focus indicators and keyboard interaction patterns
- **Error Handling**: Visual and programmatic error state communication
- **Responsive Design**: Consistent behavior across different screen sizes
- **Type Safety**: Full TypeScript support with proper type definitions

## Behavior

### Selection Behavior

- **Mutual Exclusivity**: Only one radio button can be selected within a group
- **Group Management**: RadioGroup automatically manages selection state
- **Value Tracking**: Selected value is tracked and propagated through onChange callbacks
- **Default Selection**: Support for default selected values in uncontrolled mode

### Keyboard Navigation

- **Arrow Keys**: Navigate between radio buttons within a group
- **Auto-Selection**: Arrow key navigation automatically selects the focused radio
- **Tab Behavior**: Tab moves focus between radio groups, not individual radios
- **Space/Enter**: Activates the currently focused radio button

### State Management

- **Controlled Mode**: External state management with value and onChange props
- **Uncontrolled Mode**: Internal state management with defaultValue prop
- **Individual Controls**: Can be used standalone outside of groups
- **Validation Support**: Built-in support for required and error states

### Focus Management

- **Group Focus**: Only the selected radio (or first radio) in a group is focusable
- **Focus Indicators**: Clear visual focus indicators for keyboard users
- **Focus Restoration**: Focus is properly managed during dynamic updates
- **Skip Navigation**: Proper tab order through radio groups

### Content Management

- **Label Association**: Proper label association with radio inputs
- **Subtext Support**: Optional descriptive text below main labels
- **Slot Content**: Support for additional content like badges or icons
- **Required Indicators**: Automatic asterisk display for required fields

## Accessibility

- **ARIA Compliance**: Uses proper `role="radiogroup"` and `aria-checked` attributes
- **Keyboard Navigation**: Full keyboard support with arrow keys for navigation
- **Screen Reader Support**: Proper announcements of state changes and group structure
- **Focus Management**: Focus is trapped within radio groups during navigation
- **Label Association**: Each radio is properly associated with its label
- **Group Semantics**: Radio groups are announced as groups to screen readers
- **State Communication**: Current selection and states are communicated to assistive technologies
- **Required Field Support**: Required fields are announced to screen readers
- **Error State Support**: Error states are communicated through ARIA attributes

## Best Practices

### When to Use Radio Buttons

- **Mutually Exclusive Choices**: When users must select exactly one option from a list
- **Visible Options**: When all options should be visible at once (typically 2-7 options)
- **Single Selection**: When only one choice is allowed, unlike checkboxes
- **Form Controls**: As part of forms where a selection is required

### Radio Group Design Guidelines

- **Clear Labels**: Use descriptive labels that clearly explain each option
- **Logical Ordering**: Order options logically (alphabetical, by importance, or frequency)
- **Default Selection**: Consider providing a sensible default selection
- **Group Related Options**: Keep related radio buttons together in groups

### Content Guidelines

- **Concise Labels**: Keep radio button labels brief but descriptive
- **Helpful Subtext**: Use subtext to provide additional context when needed
- **Consistent Formatting**: Maintain consistent label and subtext formatting
- **Clear Hierarchy**: Use typography and spacing to establish clear content hierarchy

### Form Integration

- **Validation**: Implement proper validation for required radio groups
- **Error Messaging**: Provide clear error messages for validation failures
- **Default Values**: Set appropriate default values for better user experience
- **State Persistence**: Consider persisting selections during form interactions
