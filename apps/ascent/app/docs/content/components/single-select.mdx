## Usage

```tsx
import { SingleSelect, SelectMenuSize, SelectMenuVariant } from 'blend-v1'
import { useState } from 'react'
import { User, MapPin, Star } from 'lucide-react'

function MyComponent() {
    const [selectedUser, setSelectedUser] = useState('')

    const teamMembers = [
        {
            groupLabel: 'Developers',
            items: [
                {
                    label: 'Sarah Johnson',
                    value: 'sarah-johnson',
                    subLabel: 'Frontend Developer',
                    slot1: <User size={16} className="text-blue-500" />,
                    slot4: <Star size={14} className="text-yellow-500" />,
                },
                {
                    label: 'Mike Chen',
                    value: 'mike-chen',
                    subLabel: 'Backend Developer',
                    slot1: <User size={16} className="text-green-500" />,
                    slot4: <Star size={14} className="text-yellow-500" />,
                },
            ],
        },
        {
            groupLabel: 'Designers',
            items: [
                {
                    label: 'Alex Rivera',
                    value: 'alex-rivera',
                    subLabel: 'UI/UX Designer',
                    slot1: <User size={16} className="text-purple-500" />,
                    slot4: <Star size={14} className="text-yellow-500" />,
                },
            ],
        },
    ]

    return (
        <SingleSelect
            label="Assign team member"
            placeholder="Select a team member"
            items={teamMembers}
            selected={selectedUser}
            onSelect={setSelectedUser}
            enableSearch={true}
            searchPlaceholder="Search team members..."
            size={SelectMenuSize.MEDIUM}
            variant={SelectMenuVariant.CONTAINER}
            required={true}
            helpIconText="Choose a team member to assign this task"
        />
    )
}
```

## Features

- **Size Variants**: Three sizes (small, medium, large) for different interface densities
- **Visual Variants**: Container and no-container styles for flexible integration
- **Search Functionality**: Optional built-in search with customizable placeholder
- **Grouped Organization**: Support for organizing items into labeled groups with separators
- **Rich Content Support**: Multiple slot positions for icons, badges, and custom elements
- **Submenu Navigation**: Hierarchical menu structure with nested item support
- **Form Integration**: Complete form support with labels, validation, and error states
- **Mobile Optimization**: Automatic drawer mode on mobile devices for better UX
- **Custom Triggers**: Support for completely custom trigger elements
- **Flexible Positioning**: 4 sides and 3 alignment options with offset controls
- **Tooltip Integration**: Built-in tooltip support for menu items with configuration options
- **Keyboard Navigation**: Full keyboard support with arrow keys and type-ahead search
- **Error Handling**: Comprehensive error state management with custom error messages
- **Accessibility First**: ARIA compliance, screen reader support, and focus management
- **Responsive Design**: Adapts to different screen sizes and device types

## Behavior

### Selection Behavior

- **Single Selection**: Only one option can be selected at a time
- **Value Management**: Selected value is controlled through props
- **Callback Execution**: onSelect callback fires when selection changes
- **Auto-Close**: Dropdown automatically closes after selection

### Search Behavior

- **Live Filtering**: Search filters options in real-time as user types
- **Label and SubLabel**: Searches both main label and subLabel text
- **Case Insensitive**: Search is not case sensitive
- **Group Filtering**: Groups with no matching items are hidden

### Keyboard Navigation

- **Arrow Keys**: Navigate through menu options
- **Enter/Space**: Select the currently focused option
- **Escape**: Close the dropdown menu
- **Type Ahead**: Jump to options by typing first letters

### Positioning Logic

- **Auto-Positioning**: Dropdown repositions to stay within viewport
- **Side Options**: Can appear on top, bottom, left, or right of trigger
- **Alignment Control**: Align to start, center, or end of trigger
- **Offset Adjustments**: Fine-tune positioning with pixel offsets

### Mobile Behavior

- **Drawer Mode**: Renders as drawer on mobile devices when useDrawerOnMobile is true
- **Touch Optimization**: Optimized for touch interactions
- **Responsive Sizing**: Adapts to mobile screen constraints

### Error State Management

- **Visual Indicators**: Error styling applied to trigger and labels
- **Error Messages**: Displays error message below the field
- **State Synchronization**: Error state clears when valid selection is made

### Focus Management

- **Trigger Focus**: Focus is managed on the trigger element
- **Menu Focus**: Focus moves through menu items during navigation
- **Focus Restoration**: Focus returns to trigger when menu closes
- **Focus Events**: onFocus and onBlur callbacks for external focus management

## Accessibility

- **ARIA Compliance**: Full ARIA support with `role="combobox"`, `aria-expanded`, and `aria-haspopup`
- **Keyboard Navigation**: Complete keyboard support with arrow keys, Enter, Escape, and Tab
- **Screen Reader Support**: Proper announcements for state changes and selections
- **Focus Management**: Focus is trapped within dropdown and restored properly
- **Label Association**: Form labels are properly associated with the select trigger
- **Group Semantics**: Menu groups are announced as groups to screen readers
- **Search Accessibility**: Search functionality is accessible with proper ARIA attributes
- **Error Communication**: Error states are communicated through ARIA attributes
- **Required Field Support**: Required fields are announced to assistive technologies

## Best Practices

### When to Use SingleSelect

- **Option Selection**: When users need to choose one option from multiple choices
- **Form Fields**: As form controls where a specific selection is required
- **Settings**: For configuration options and preferences
- **Data Filtering**: For filtering or sorting data based on categories

### Design Guidelines

- **Clear Labels**: Use descriptive labels that clearly explain the purpose
- **Logical Grouping**: Group related options together with meaningful group labels
- **Appropriate Sizing**: Choose size based on interface density and importance
- **Consistent Patterns**: Use consistent selection patterns throughout your application

### Content Guidelines

- **Concise Options**: Keep option labels brief but descriptive
- **Helpful SubLabels**: Use subLabels to provide additional context when needed
- **Logical Ordering**: Order options logically (alphabetical, by frequency, or importance)
- **Search Support**: Enable search for lists with many options

### Form Integration

- **Validation**: Implement proper validation with clear error messages
- **Required Fields**: Mark required fields clearly with asterisks
- **Help Text**: Provide contextual help when selection criteria may be unclear
- **Default Values**: Consider providing sensible default selections when appropriate

### Mobile Considerations

- **Drawer Mode**: Use drawer mode on mobile for better touch experience
- **Touch Targets**: Ensure adequate touch target sizes for mobile devices
- **Search Functionality**: Enable search for long option lists on mobile
- **Performance**: Consider lazy loading for very large option sets
