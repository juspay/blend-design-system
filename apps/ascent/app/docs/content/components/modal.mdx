## Usage

```tsx
import { Modal, ButtonType } from 'blend-v1'
import { useState } from 'react'

function MyComponent() {
    const [isOpen, setIsOpen] = useState(false)

    return (
        <>
            <button onClick={() => setIsOpen(true)}>Open Modal</button>

            <Modal
                isOpen={isOpen}
                onClose={() => setIsOpen(false)}
                title="Confirm Action"
                subtitle="This action cannot be undone. Please confirm to proceed."
                primaryAction={{
                    text: 'Confirm',
                    buttonType: ButtonType.PRIMARY,
                    onClick: () => {
                        console.log('Action confirmed')
                        setIsOpen(false)
                    },
                }}
                secondaryAction={{
                    text: 'Cancel',
                    buttonType: ButtonType.SECONDARY,
                    onClick: () => setIsOpen(false),
                }}
                showCloseButton={true}
                closeOnBackdropClick={true}
            >
                <div className="space-y-4">
                    <p>
                        Are you sure you want to delete this item? This action
                        will permanently remove the selected data.
                    </p>
                    <p>
                        Please review your selection and confirm if you wish to
                        proceed.
                    </p>
                </div>
            </Modal>
        </>
    )
}
```

## Features

- **Overlay Management**: Full-screen overlay with backdrop blur and click-to-close options
- **Responsive Design**: Automatic mobile drawer mode for better mobile experience
- **Flexible Actions**: Primary and secondary actions with full Button component configuration
- **Header Right Slot**: Additional content displayed in the header right area
- **Scroll Lock**: Automatically prevents body scroll when modal is open
- **Accessibility First**: Full keyboard navigation, focus management, and ARIA compliance
- **Animation Support**: Smooth enter/exit animations with backdrop transitions
- **Header Control**: Show/hide dividers and close button as needed
- **Size Control**: Configurable minimum width and responsive sizing
- **Mobile Optimization**: Drawer-style presentation on mobile devices
- **Backdrop Interaction**: Configurable backdrop click behavior

## Behavior

### Modal Lifecycle

- **Open Animation**: Smooth fade-in animation for backdrop and modal content
- **Close Animation**: Fade-out animation when closing
- **Scroll Lock**: Body scroll is locked when modal is open
- **Focus Management**: Focus is trapped within the modal

### Interaction Patterns

- **Backdrop Click**: Configurable backdrop click-to-close behavior
- **Escape Key**: Escape key closes the modal by default
- **Close Button**: Optional X button in header for closing
- **Action Buttons**: Primary and secondary actions in footer

### Mobile Behavior

- **Responsive Layout**: Adapts to smaller screens automatically
- **Drawer Mode**: Optional drawer-style presentation on mobile
- **Touch Interactions**: Optimized for touch interactions
- **Safe Areas**: Respects mobile safe areas and notches

### Content Management

- **Scrollable Content**: Content area scrolls when content exceeds viewport
- **Header Pinning**: Header stays fixed when content scrolls
- **Footer Pinning**: Footer stays fixed at bottom
- **Responsive Sizing**: Modal adapts to content size with constraints

## Accessibility

- **Focus Management**: Focus is trapped within modal and restored on close
- **Keyboard Navigation**: Full keyboard support with Tab, Shift+Tab, and Escape
- **Screen Reader Support**: Proper ARIA roles, labels, and live regions
- **ARIA Compliance**: Uses `role="dialog"`, `aria-modal`, and `aria-labelledby`
- **High Contrast**: Supports high contrast mode and custom color schemes
- **Motion Preferences**: Respects user's reduced motion preferences

## Best Practices

### When to Use Modals

- **Important Decisions**: For critical actions that need user confirmation
- **Form Dialogs**: For forms that interrupt the main workflow
- **Information Display**: For detailed information that needs focus
- **Error Messages**: For critical error messages that need attention

### Modal Design Guidelines

- **Clear Purpose**: Modal should have a clear, single purpose
- **Concise Content**: Keep content focused and concise
- **Clear Actions**: Action buttons should clearly indicate what will happen
- **Escape Routes**: Always provide a way to close the modal

### Mobile Considerations

- **Use Drawer Mode**: Enable drawer mode for better mobile experience
- **Touch Targets**: Ensure buttons are large enough for touch interaction
- **Content Length**: Keep content length appropriate for mobile screens
- **Safe Areas**: Consider mobile safe areas when positioning content

### Action Button Guidelines

- **Primary Action**: Use for the main action user should take
- **Secondary Action**: Use for alternative or cancel actions
- **Dangerous Actions**: Use danger button type for destructive actions
- **Loading States**: Show loading state for async operations
