## Usage

```tsx
import {
    Menu,
    MenuAlignment,
    MenuSide,
    MenuItemVariant,
    MenuItemActionType,
} from 'blend-v1'
import { Button, ButtonType } from 'blend-v1'
import { Settings, User, LogOut, ChevronDown } from 'lucide-react'

function MyComponent() {
    const menuItems = [
        {
            label: 'Account',
            items: [
                {
                    label: 'Profile',
                    subLabel: 'Manage your profile',
                    slot1: <User size={16} />,
                    onClick: () => console.log('Profile clicked'),
                },
                {
                    label: 'Settings',
                    subLabel: 'Configure preferences',
                    slot1: <Settings size={16} />,
                    onClick: () => console.log('Settings clicked'),
                },
            ],
        },
        {
            label: 'Actions',
            items: [
                {
                    label: 'Sign Out',
                    slot1: <LogOut size={16} />,
                    variant: MenuItemVariant.ACTION,
                    actionType: MenuItemActionType.DANGER,
                    onClick: () => console.log('Sign out clicked'),
                },
            ],
            showSeparator: true,
        },
    ]

    return (
        <Menu
            trigger={
                <Button
                    text="Account Menu"
                    buttonType={ButtonType.SECONDARY}
                    trailingIcon={<ChevronDown size={16} />}
                />
            }
            items={menuItems}
            alignment={MenuAlignment.END}
            side={MenuSide.BOTTOM}
            enableSearch={true}
            searchPlaceholder="Search menu..."
        />
    )
}
```

## Features

- **Flexible Positioning**: 4 sides and 3 alignment options for optimal placement
- **Grouped Items**: Organize menu items into labeled groups with optional separators
- **Search Functionality**: Built-in search to filter menu items by label
- **Multiple Content Slots**: 4 content slots per item for icons, badges, and additional content
- **Action Variants**: Special styling for primary and danger actions
- **Submenu Support**: Nested menu items for hierarchical navigation
- **Controlled/Uncontrolled**: Support for both controlled and uncontrolled open states
- **Modal Mode**: Renders as modal on mobile devices for better UX
- **Collision Detection**: Automatic repositioning to stay within viewport
- **Disabled States**: Individual menu items can be disabled
- **Custom Sizing**: Configurable width and height constraints
- **Virtual Scrolling**: Performance optimization for large lists
- **Accessibility First**: Full keyboard navigation and screen reader support

## Behavior

### Positioning Logic

- **Side**: Menu appears on specified side of trigger (top, right, bottom, left)
- **Alignment**: Menu aligns to start, center, or end of trigger
- **Collision Detection**: Automatically repositions if menu would overflow viewport
- **Offsets**: Fine-tune positioning with side and align offsets

### Search Functionality

- **Live Filtering**: Menu items filter as user types
- **Label Matching**: Searches both main label and subLabel text
- **Case Insensitive**: Search is not case sensitive
- **Group Filtering**: Empty groups are hidden when all items are filtered out

### Interaction Patterns

- **Click to Open**: Trigger click opens the menu
- **Click Outside**: Clicking outside closes the menu
- **Escape Key**: Escape key closes the menu
- **Item Selection**: Clicking menu item executes action and closes menu

### Mobile Behavior

- **Modal Mode**: When `asModal={true}`, renders as full-screen modal on mobile
- **Touch Interactions**: Optimized for touch interactions
- **Responsive Sizing**: Adapts to screen size constraints

## Accessibility

- **Keyboard Navigation**: Full keyboard support with arrow keys, Enter, and Escape
- **Focus Management**: Proper focus management when opening/closing menu
- **Screen Reader Support**: ARIA labels and proper semantic structure
- **ARIA Compliance**: Uses `aria-expanded`, `aria-haspopup`, and `role` attributes
- **High Contrast**: Supports high contrast mode and custom color schemes
- **Tab Navigation**: Proper tab order through menu items

## Best Practices

### Menu Organization

- **Group Related Items**: Use groups to organize related functionality
- **Logical Order**: Place most common actions at the top
- **Dangerous Actions**: Use danger action type for destructive actions
- **Clear Labels**: Use descriptive labels that clearly indicate the action

### Search Implementation

- **Enable for Long Lists**: Use search when menu has many items
- **Meaningful Placeholder**: Provide helpful search placeholder text
- **Consider Performance**: Search filters client-side, suitable for moderate lists

### Positioning Guidelines

- **Consider Context**: Choose side and alignment based on trigger position
- **Avoid Overflow**: Ensure menu doesn't extend beyond viewport
- **Mobile Considerations**: Use modal mode for complex menus on mobile
