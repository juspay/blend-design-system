## Usage

```tsx
import { TextInput } from '@juspay/blend-design-system'

function MyComponent() {
    const [value, setValue] = useState('')

    return (
        <TextInput
            label="Email Address"
            placeholder="Enter your email"
            value={value}
            onChange={(e) => setValue(e.target.value)}
            hintText="We'll never share your email"
        />
    )
}
```

- **sm**: Mobile-optimized with floating label behavior for large size
- **lg**: Desktop layout with consistent behavior across all sizes

## Features

- **Responsive Design**: Adaptive layout with floating labels on mobile for large size
- **Flexible Slots**: Left and right slots for icons, buttons, or custom content
- **Validation States**: Built-in error styling with custom error messages
- **Accessibility**: Full keyboard navigation and screen reader support
- **Form Integration**: Native HTML input attributes and form compatibility
- **Visual States**: Hover, focus, and disabled states with smooth transitions
- **Customizable Sizing**: Three size variants for different layout needs
- **Helper Text**: Support for hints, help icons, and error messages
- **Dynamic Padding**: Automatic padding adjustment based on slot content

## Usage Examples

### Basic Input

Simple text input with label and hint text:

```tsx
import { TextInput } from '@juspay/blend-design-system'

const [email, setEmail] = useState('')

<TextInput
    label="Email Address"
    placeholder="Enter your email"
    value={email}
    onChange={(e) => setEmail(e.target.value)}
    hintText="We'll never share your email with anyone"
/>
```

### Input with Error State

Text input showing validation error:

```tsx
const [password, setPassword] = useState('')
const hasError = password.length > 0 && password.length < 8

<TextInput
    label="Password"
    placeholder="Enter password"
    value={password}
    onChange={(e) => setPassword(e.target.value)}
    error={hasError}
    errorMessage="Password must be at least 8 characters long"
    required
/>
```

### Input with Slots

Text input with icon slots for enhanced UX:

```tsx
import { Search, Eye, EyeOff } from 'lucide-react'

const [searchQuery, setSearchQuery] = useState('')
const [showPassword, setShowPassword] = useState(false)

// Search input with left icon
<TextInput
    placeholder="Search..."
    value={searchQuery}
    onChange={(e) => setSearchQuery(e.target.value)}
    leftSlot={<Search size={16} />}
/>

// Password input with toggle visibility
<TextInput
    label="Password"
    type={showPassword ? 'text' : 'password'}
    value={password}
    onChange={(e) => setPassword(e.target.value)}
    rightSlot={
        <button
            type="button"
            onClick={() => setShowPassword(!showPassword)}
        >
            {showPassword ? <EyeOff size={16} /> : <Eye size={16} />}
        </button>
    }
/>
```

## Behavior

### Responsive Floating Labels

- **Large Size on Mobile**: Automatically shows floating label animation
- **Label Positioning**: Smooth transition between placeholder and floating states
- **Touch Optimization**: Enhanced mobile experience with larger touch targets

### Focus Management

- **Focus States**: Visual feedback with outline and shadow changes
- **Keyboard Navigation**: Full tab navigation support
- **Focus Callbacks**: Custom onFocus and onBlur handlers

### Dynamic Layout

- **Slot Integration**: Automatic padding adjustment for left/right content
- **Width Calculation**: Dynamic measurement of slot content for precise spacing
- **Responsive Tokens**: Breakpoint-aware styling and spacing

### Validation Integration

- **Error Styling**: Automatic error state styling when error prop is true
- **Message Display**: Error messages shown below input with proper styling
- **State Consistency**: Visual state matches validation state throughout interaction

## Best Practices

### When to Use TextInput

- **Form Fields**: Primary choice for text-based form inputs
- **Search Interfaces**: With appropriate left slot icons
- **Data Entry**: For capturing user information and content
- **Settings Panels**: For configurable text values

### Label and Hint Guidelines

- **Clear Labels**: Use descriptive labels that explain the expected input
- **Helpful Hints**: Provide context or format examples in hint text
- **Error Messages**: Write specific, actionable error messages
- **Required Indication**: Use required prop and clear visual indicators

### Size Selection

- **Small**: Use in compact forms or secondary inputs
- **Medium**: Default choice for most form scenarios
- **Large**: Use for primary inputs or mobile-first experiences

### Slot Usage

- **Left Slots**: Icons, currency symbols, or input prefixes
- **Right Slots**: Action buttons, validation icons, or suffixes
- **Interactive Content**: Ensure slot content has proper focus management
- **Visual Balance**: Keep slot content appropriately sized

### Accessibility Considerations

- **Label Association**: Always provide labels for screen readers
- **Error Announcement**: Ensure error messages are announced to screen readers
- **Focus Indicators**: Maintain visible focus indicators for keyboard users
- **Form Structure**: Use proper form markup and validation patterns

### Form Integration

- **Controlled Components**: Always use controlled inputs with value and onChange
- **Validation Timing**: Validate on blur for better user experience
- **Form Libraries**: Compatible with React Hook Form, Formik, and other libraries
- **Submit Handling**: Integrate with form submission and validation flows
