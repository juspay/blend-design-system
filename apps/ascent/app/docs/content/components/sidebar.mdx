## Usage

```tsx
import { Sidebar } from 'blend-v1'
import { Dashboard, Projects, Settings } from 'lucide-react'

function MyComponent() {
    const directoryData = [
        {
            id: '1',
            label: 'Dashboard',
            href: '/dashboard',
            icon: <Dashboard size={16} />,
        },
        {
            id: '2',
            label: 'Projects',
            href: '/projects',
            icon: <Projects size={16} />,
            children: [
                {
                    id: '2-1',
                    label: 'Active Projects',
                    href: '/projects/active',
                },
                {
                    id: '2-2',
                    label: 'Completed',
                    href: '/projects/completed',
                },
            ],
        },
        {
            id: '3',
            label: 'Settings',
            href: '/settings',
            icon: <Settings size={16} />,
        },
    ]

    return (
        <Sidebar
            data={directoryData}
            topbar={
                <div className="flex items-center justify-between w-full">
                    <h1 className="text-lg font-semibold">My Application</h1>
                    <button className="px-3 py-1 bg-gray-100 rounded">
                        Profile
                    </button>
                </div>
            }
            footer={
                <div className="flex items-center gap-2">
                    <div className="w-8 h-8 bg-gray-300 rounded-full" />
                    <span className="text-sm">John Doe</span>
                </div>
            }
        >
            <div className="p-6">
                <h2 className="text-xl font-bold mb-4">Main Content</h2>
                <p>Your main application content goes here.</p>
            </div>
        </Sidebar>
    )
}
```

## Features

- **Collapsible Sidebar**: Expandable/collapsible sidebar with smooth animations and hover states
- **Directory Navigation**: Hierarchical navigation with nested menu support using Directory component
- **Dual Panel Layout**: Optional secondary left panel for tenant or context switching
- **Keyboard Shortcuts**: Configurable keyboard shortcut for sidebar toggle (default: '/')
- **Responsive Design**: Mobile-friendly with auto-collapse and hover expansion
- **Integrated Topbar**: Sticky topbar with backdrop blur and flexible content area
- **Footer Support**: Optional footer area with user information or custom content
- **Merchant Selection**: Built-in merchant/context switcher with dropdown selection
- **Overflow Handling**: Smart overflow management for secondary panel items
- **Smooth Animations**: CSS-based transitions for expand/collapse and hover states
- **Customizable Header**: Flexible sidebar header area with custom slot support
- **Backdrop Effects**: Modern backdrop blur effects for enhanced visual hierarchy
- **Auto-scrolling**: Custom scrollbar styling with overflow management
- **State Management**: Internal state management for expand/collapse functionality

### Dual Panel Sidebar

Sidebar with secondary tenant/context panel

```tsx
import { Building, Users } from 'lucide-react'

const [selectedTenant, setSelectedTenant] = useState('company-a')

const leftPanelConfig = {
    items: [
        {
            label: 'Company A',
            icon: <Building size={16} />,
            value: 'company-a'
        },
        {
            label: 'Company B',
            icon: <Building size={16} />,
            value: 'company-b'
        },
        {
            label: 'Company C',
            icon: <Users size={16} />,
            value: 'company-c'
        }
    ],
    selected: selectedTenant,
    onSelect: setSelectedTenant,
    maxVisibleItems: 2
}

<Sidebar
    data={navigationData}
    leftPanel={leftPanelConfig}
    topbar={
        <div className="flex items-center gap-3">
            <h1>Multi-Tenant App</h1>
            <span className="text-sm text-gray-500">
                ({selectedTenant})
            </span>
        </div>
    }
>
    <div className="p-6">
        <h2>Tenant-Specific Content</h2>
        <p>Content for {selectedTenant}</p>
    </div>
</Sidebar>
```

## Behavior

### Collapse/Expand System

- **Default State**: Sidebar starts in expanded state
- **Toggle Methods**: Click toggle button or use keyboard shortcut
- **Hover Expansion**: Collapsed sidebar expands on hover
- **Animation**: Smooth CSS transitions for state changes
- **Persistence**: State maintained during session

### Navigation Interaction

- **Directory Integration**: Uses Directory component for navigation rendering
- **Nested Menus**: Supports hierarchical navigation with expandable sub-menus
- **Active States**: Visual indicators for current page/section
- **Click Handling**: Standard link navigation with href support

### Responsive Design

- **Mobile Adaptation**: Automatic behavior adjustment for mobile screens
- **Touch Support**: Touch-friendly interactions for mobile devices
- **Overflow Management**: Smart content overflow handling
- **Backdrop Effects**: Modern backdrop blur for depth perception

### Panel Management

- **Dual Panel Support**: Optional secondary panel for context switching
- **Overflow Handling**: Dropdown menu for excess panel items
- **Selection States**: Visual feedback for selected panel items
- **Dynamic Content**: Reactive updates based on panel selection

## Best Practices

### When to Use Sidebar

- **Complex Applications**: For applications with multiple sections and deep navigation
- **Dashboard Layouts**: For admin panels and data-heavy interfaces
- **Multi-Context Apps**: For applications with tenant or workspace switching
- **Content Management**: For systems with hierarchical content organization

### Navigation Design

- **Clear Hierarchy**: Organize navigation items in logical groups
- **Consistent Icons**: Use consistent icon styles throughout navigation
- **Meaningful Labels**: Use clear, descriptive labels for navigation items
- **Logical Grouping**: Group related navigation items together

### Layout Considerations

- **Content Area**: Ensure main content area adapts to sidebar state
- **Mobile Experience**: Design for mobile-first with collapsible behavior
- **Performance**: Optimize navigation data structure for large menus
- **State Management**: Handle navigation state at application level

### Accessibility Considerations

- **Keyboard Navigation**: Ensure all functionality works with keyboard
- **Screen Readers**: Provide appropriate ARIA labels and roles
- **Focus Management**: Maintain logical focus order throughout navigation
- **Color Independence**: Don't rely solely on color for navigation states

### Customization Guidelines

- **Brand Integration**: Customize header area with brand elements
- **Consistent Styling**: Maintain visual consistency with application theme
- **Footer Content**: Use footer for user information or secondary actions
- **Responsive Breakpoints**: Consider custom breakpoints for sidebar behavior
