## Usage

```tsx
import { Popover, PopoverSize, Button, ButtonType } from 'blend-v1'
import { Settings } from 'lucide-react'
import { useState } from 'react'

function MyComponent() {
    const [isOpen, setIsOpen] = useState(false)

    return (
        <Popover
            heading="Account Settings"
            description="Manage your account preferences and settings"
            trigger={
                <Button
                    text="Settings"
                    buttonType={ButtonType.SECONDARY}
                    leadingIcon={<Settings size={16} />}
                />
            }
            open={isOpen}
            onOpenChange={setIsOpen}
            size={PopoverSize.MEDIUM}
            side="bottom"
            align="center"
            primaryAction={{
                text: 'Save Changes',
                buttonType: ButtonType.PRIMARY,
                onClick: () => {
                    console.log('Settings saved')
                    setIsOpen(false)
                },
            }}
            secondaryAction={{
                text: 'Cancel',
                buttonType: ButtonType.SECONDARY,
                onClick: () => setIsOpen(false),
            }}
            showCloseButton={true}
        >
            <div className="space-y-4">
                <div>
                    <label className="block text-sm font-medium mb-1">
                        Display Name
                    </label>
                    <input
                        type="text"
                        placeholder="Enter your display name"
                        className="w-full px-3 py-2 border rounded-md"
                    />
                </div>
                <div>
                    <label className="flex items-center">
                        <input type="checkbox" className="mr-2" />
                        Enable email notifications
                    </label>
                </div>
            </div>
        </Popover>
    )
}
```

## Features

- **Flexible Positioning**: 4 sides (top, right, bottom, left) with 3 alignment options (start, center, end)
- **Size Variants**: Two size options (small, medium) for different content types
- **Action Buttons**: Primary and secondary action buttons with full Button component support
- **Header Management**: Optional header with title and description text
- **Close Button**: Optional X button for dismissing the popover
- **Controlled State**: Support for both controlled and uncontrolled open states
- **Modal Behavior**: Option to render as modal dialog for critical interactions
- **Custom Dimensions**: Configurable width, height, and size constraints
- **Mobile Optimization**: Automatic drawer mode on mobile devices for better UX
- **Shadow Options**: 7 shadow variants for different visual emphasis levels
- **Built on Radix UI**: Leverages Radix UI Popover for robust accessibility and positioning
- **Collision Detection**: Automatic repositioning to stay within viewport boundaries
- **Custom Styling**: Comprehensive component token system for styling customization
- **Responsive Design**: Adapts to different screen sizes and device types
- **Accessibility First**: Full keyboard navigation, focus management, and screen reader support

## Usage Examples

### Basic Information Popover

Simple popover displaying contextual information

```tsx
import { Popover, PopoverSize, Button, ButtonType } from 'blend-v1'
import { Info } from 'lucide-react'
;<Popover
    heading="Feature Information"
    description="Learn more about this feature"
    trigger={
        <Button
            text="Learn More"
            buttonType={ButtonType.SECONDARY}
            leadingIcon={<Info size={16} />}
        />
    }
    size={PopoverSize.MEDIUM}
    side="bottom"
    align="center"
>
    <div className="space-y-3">
        <p>This feature helps you manage your workspace more efficiently.</p>
        <ul className="list-disc list-inside space-y-1 text-sm">
            <li>Organize your projects</li>
            <li>Collaborate with team members</li>
            <li>Track progress in real-time</li>
        </ul>
    </div>
</Popover>
```

### Controlled State Popover

Popover with external state management

```tsx
import { Settings } from 'lucide-react'

const [isSettingsOpen, setIsSettingsOpen] = useState(false)
const [theme, setTheme] = useState('light')

<Popover
    heading="Appearance Settings"
    description="Customize your interface"
    trigger={
        <Button
            text="Settings"
            buttonType={ButtonType.SECONDARY}
            leadingIcon={<Settings size={16} />}
        />
    }
    open={isSettingsOpen}
    onOpenChange={setIsSettingsOpen}
    primaryAction={{
        text: 'Apply',
        buttonType: ButtonType.PRIMARY,
        onClick: () => {
            console.log('Settings applied:', { theme })
            setIsSettingsOpen(false)
        },
    }}
    secondaryAction={{
        text: 'Reset',
        buttonType: ButtonType.SECONDARY,
        onClick: () => setTheme('light'),
    }}
>
    <div className="space-y-4">
        <div>
            <label className="block text-sm font-medium mb-2">Theme</label>
            <div className="space-y-2">
                {['light', 'dark', 'auto'].map((option) => (
                    <label key={option} className="flex items-center">
                        <input
                            type="radio"
                            name="theme"
                            value={option}
                            checked={theme === option}
                            onChange={(e) => setTheme(e.target.value)}
                            className="mr-2"
                        />
                        <span className="text-sm capitalize">{option}</span>
                    </label>
                ))}
            </div>
        </div>
    </div>
</Popover>
```

## Behavior

### Positioning Logic

- **Side Placement**: Popover appears on specified side of trigger (top, right, bottom, left)
- **Alignment Options**: Aligns to start, center, or end of trigger element
- **Collision Detection**: Automatically repositions if popover would overflow viewport
- **Offset Controls**: Fine-tune positioning with side and align offsets

### Interaction Patterns

- **Click to Open**: Trigger click opens the popover
- **Click Outside**: Clicking outside the popover closes it
- **Escape Key**: Escape key closes the popover
- **Action Execution**: Action buttons execute their onClick handlers and close popover

### State Management

- **Controlled Mode**: Use `open` and `onOpenChange` props for external state control
- **Uncontrolled Mode**: Component manages its own open/closed state internally
- **Close Events**: Multiple ways to close including actions, close button, escape key, and backdrop click

### Mobile Behavior

- **Drawer Mode**: When `useDrawerOnMobile` is true, renders as drawer on mobile devices
- **Touch Interactions**: Optimized for touch interactions and gestures
- **Responsive Sizing**: Adapts to mobile screen constraints and safe areas

### Content Management

- **Flexible Content**: Supports any React content in children
- **Header Control**: Optional header with title and description
- **Footer Actions**: Optional action buttons in footer area
- **Custom Padding**: Automatic padding handling with override options for custom content

## Accessibility

- **Focus Management**: Focus is trapped within popover when open and restored on close
- **Keyboard Navigation**: Full keyboard support with Tab, Shift+Tab, and Escape keys
- **Screen Reader Support**: Proper ARIA roles, labels, and live regions
- **ARIA Compliance**: Uses `aria-expanded`, `aria-haspopup`, and proper dialog semantics
- **High Contrast**: Supports high contrast mode and respects user color preferences
- **Motion Preferences**: Respects user's reduced motion preferences for animations

## Best Practices

### When to Use Popovers

- **Contextual Information**: For displaying additional information without navigation
- **Quick Actions**: For forms or actions that don't require full page navigation
- **Settings and Preferences**: For configuration options that are secondary to main flow
- **Help and Documentation**: For providing contextual help or feature explanations

### Popover Design Guidelines

- **Clear Purpose**: Each popover should have a focused, single purpose
- **Appropriate Trigger**: Use clear, descriptive triggers that indicate popover content
- **Concise Content**: Keep content focused and avoid overwhelming users
- **Consistent Positioning**: Use consistent positioning patterns throughout your application

### Content Guidelines

- **Scannable Layout**: Organize content with clear hierarchy and spacing
- **Action Clarity**: Make action buttons clearly indicate what will happen
- **Error Handling**: Provide clear feedback for form validation and errors
- **Loading States**: Show loading states for async operations in actions

### Mobile Considerations

- **Use Drawer Mode**: Enable drawer mode for better mobile experience
- **Touch Targets**: Ensure interactive elements meet minimum touch target sizes
- **Content Length**: Keep content appropriate for mobile screen sizes
- **Safe Areas**: Consider mobile safe areas and keyboard interactions
