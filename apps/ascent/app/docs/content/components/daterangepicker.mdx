## Usage

```tsx
import { DateRangePicker, DateRange } from '@juspay/blend-design-system'

function MyComponent() {
    const [dateRange, setDateRange] = useState<DateRange | undefined>(undefined)

    const handleDateRangeChange = (range: DateRange) => {
        setDateRange(range)
        console.log('Selected date range:', range)
    }

    return (
        <DateRangePicker
            value={dateRange}
            onChange={handleDateRangeChange}
            showDateTimePicker={true}
            showPresets={true}
            allowSingleDateSelection={false}
            dateFormat="dd/MM/yyyy"
        />
    )
}
```

## Behavior

### Date Selection Flow

- **Single Click**: Select single date or start of range
- **Range Selection**: Click start date, then end date to complete range
- **Visual Feedback**: Highlight selected dates and range preview
- **Validation**: Ensure end date is after start date

### Mobile Experience

- **Drawer Interface**: Full-screen drawer on mobile devices
- **Touch Optimized**: Large touch targets for mobile interaction
- **Preset Integration**: Quick access to common date ranges
- **Time Picker**: Scrollable time selection interface

### Keyboard Navigation

- **Tab Navigation**: Navigate between inputs and calendar
- **Arrow Keys**: Navigate calendar grid
- **Enter/Space**: Select dates and activate controls
- **Escape**: Close picker without selection

### Preset Functionality

- **Quick Selection**: Common date ranges (Today, Last 7 days, etc.)
- **Custom Presets**: Support for custom preset configurations
- **Smart Defaults**: Intelligent preset suggestions based on context
- **Mobile Adaptation**: Streamlined presets for mobile interfaces

## Best Practices

### When to Use DateRangePicker

- **Analytics Dashboards**: Date range selection for data analysis
- **Booking Systems**: Hotel, flight, and event booking date selection
- **Report Generation**: Selecting time periods for reports
- **Scheduling**: Meeting and event scheduling interfaces
- **Data Filtering**: Filtering datasets by date ranges

### Mobile Considerations

- **Drawer Mode**: Use `useDrawerOnMobile={true}` for better mobile UX
- **Preset Optimization**: Consider `skipQuickFiltersOnMobile` for simpler mobile UI
- **Touch Targets**: Ensure adequate spacing for touch interaction
- **Responsive Design**: Test across different mobile screen sizes

### Date Range Configuration

- **Logical Constraints**: Set appropriate min/max dates for your use case
- **Future/Past Restrictions**: Use `disableFutureDates` or `disablePastDates` as needed
- **Format Consistency**: Use consistent date formats across your application
- **Time Zone Handling**: Consider time zone implications for your users

### Accessibility Guidelines

- **Label Association**: Provide clear labels for screen readers
- **Keyboard Support**: Ensure full keyboard navigation functionality
- **Focus Management**: Maintain proper focus flow through the interface
- **Error Communication**: Provide accessible error messages for invalid selections
- **ARIA Labels**: Use appropriate ARIA labels for complex calendar interactions

### Performance Optimization

- **State Management**: Use appropriate state management for complex applications
- **Debouncing**: Consider debouncing for expensive operations triggered by date changes
- **Memory Management**: Clean up event listeners and timers properly
- **Lazy Loading**: Consider lazy loading for large calendar ranges

### Form Integration

- **Controlled Components**: Always use controlled date range state
- **Validation**: Integrate with form validation libraries
- **Submission Handling**: Properly format dates for API submission
- **Reset Functionality**: Implement proper form reset behavior
- **Default Values**: Provide sensible default date ranges when appropriate

### User Experience Guidelines

- **Clear Labels**: Use descriptive labels and placeholders
- **Visual Feedback**: Show clear selection states and hover effects
- **Error Handling**: Provide helpful error messages for invalid selections
- **Loading States**: Show loading indicators during data fetching
- **Confirmation**: Consider confirmation steps for critical date selections

### Data Handling

- **Time Zone Consistency**: Handle time zones consistently across your application
- **Date Formatting**: Format dates appropriately for display and storage
- **Range Validation**: Validate date ranges on both client and server
- **Edge Cases**: Handle edge cases like leap years and daylight saving time
- **Persistence**: Consider persisting user's preferred date ranges
