## Usage

```tsx
import { DropdownInput, DropdownPosition } from '@juspay/blend-design-system'

function MyComponent() {
    const [amount, setAmount] = useState('')
    const [currency, setCurrency] = useState('USD')

    const currencyOptions = [
        {
            group: 'Popular Currencies',
            items: [
                { id: 'USD', label: 'USD', value: 'USD' },
                { id: 'EUR', label: 'EUR', value: 'EUR' },
                { id: 'GBP', label: 'GBP', value: 'GBP' },
            ],
        },
    ]

    return (
        <DropdownInput
            label="Amount"
            placeholder="Enter amount"
            value={amount}
            onChange={(e) => setAmount(e.target.value)}
            dropDownValue={currency}
            onDropDownChange={setCurrency}
            dropDownItems={currencyOptions}
            dropdownPosition={DropdownPosition.RIGHT}
            hintText="Select currency and enter amount"
        />
    )
}
```

## Features

- **Dual Input System**: Combined text input and dropdown selection in one component
- **Flexible Positioning**: Dropdown can be positioned on left or right side
- **Responsive Design**: Adaptive layout with floating labels on mobile for large size
- **Integrated SingleSelect**: Uses SingleSelect component for dropdown functionality
- **Dynamic Spacing**: Automatic padding adjustment based on dropdown and slot content
- **Validation States**: Built-in error styling with custom error messages
- **Form Integration**: Native HTML input attributes and form compatibility
- **Accessibility**: Full keyboard navigation and screen reader support
- **State Management**: Coordinated state between input and dropdown components

## Behavior

### Responsive Layout

- **Large Size on Mobile**: Floating label animation for enhanced mobile experience
- **Dynamic Padding**: Automatic adjustment based on dropdown and slot positions
- **Touch Optimization**: Enhanced mobile interactions for both input and dropdown

### Dropdown Integration

- **SingleSelect Component**: Uses internal SingleSelect for dropdown functionality
- **NO_CONTAINER Variant**: Seamless integration with minimal visual separation
- **Inline Mode**: Dropdown rendered inline with input field
- **Alignment Control**: Precise positioning relative to input boundaries

### State Coordination

- **Independent Controls**: Input and dropdown maintain separate values and callbacks
- **Coordinated Focus**: Proper focus management between input and dropdown
- **Error Handling**: Unified error state affects both input and dropdown styling
- **Disabled State**: Disabling affects both components consistently

### Dynamic Spacing

- **Width Calculation**: Automatic measurement of dropdown width for padding adjustment
- **Gap Management**: Consistent spacing between input content and dropdown
- **Position Adaptation**: Padding adjusts based on left or right dropdown positioning

## Best Practices

### When to Use DropdownInput

- **Currency Inputs**: Amount entry with currency selection
- **Unit Values**: Measurements with unit selection (px, rem, %, etc.)
- **Phone Numbers**: Number entry with country code selection
- **Prefixed/Suffixed Data**: Any input requiring both text and categorical selection

### Dropdown Item Structure

- **Clear Labels**: Use descriptive labels that are easy to scan
- **Logical Grouping**: Group related items together with clear group names
- **Consistent Formatting**: Maintain consistent label format across items
- **Appropriate Icons**: Use icons or symbols when they add clarity

### Position Selection

- **Left Position**: Use for prefixes like country codes, protocols, or categories
- **Right Position**: Use for suffixes like currencies, units, or file extensions
- **User Expectations**: Follow conventional patterns for your domain

### Form Integration

- **Controlled Components**: Always use controlled inputs with proper state management
- **Validation**: Validate both input value and dropdown selection
- **Error Messages**: Provide clear error messages for validation failures
- **Required Fields**: Clearly indicate when both input and dropdown are required

### Accessibility Considerations

- **Label Association**: Ensure proper labeling for both input and dropdown
- **Keyboard Navigation**: Test tab order and arrow key navigation
- **Screen Readers**: Verify that both components are properly announced
- **Focus Indicators**: Maintain visible focus indicators for keyboard users

### Performance Optimization

- **Item Lists**: Optimize large dropdown item lists with appropriate grouping
- **State Updates**: Debounce input changes if triggering expensive operations
- **Memory Management**: Properly clean up event listeners and refs
- **Render Optimization**: Use React.memo for dropdown items if needed
