# Getting Started with Blend Design System

Welcome to Blend Design System! This guide will help you get started with implementing Blend in your project, whether you're a team member or an open source contributor.

## What is Blend?

Blend is a comprehensive design system that provides:

- **React Components**: Production-ready components with TypeScript support
- **Design Tokens**: Consistent design tokens for colors, typography, spacing, and more
- **Customizable Theming**: Flexible design tokens and theme system

## Installation

Choose your preferred package manager to install Blend:

```bash
# npm
npm install @juspay/blend-design-system

# yarn
yarn add @juspay/blend-design-system

# pnpm
pnpm add @juspay/blend-design-system
```

## Basic Setup

### 1. Wrap Your App with ThemeProvider

Wrap your application with the `ThemeProvider` to enable theming:

```tsx
import { ThemeProvider } from '@juspay/blend-design-system'

function App() {
    return (
        <ThemeProvider>
            <YourAppContent />
        </ThemeProvider>
    )
}
```

### 2. Start Using Components

Import and use components in your application:

```tsx
import { Button, ButtonType, ButtonSize } from '@juspay/blend-design-system'
import { Alert, AlertVariant, AlertStyle } from '@juspay/blend-design-system'
import { TextInput } from '@juspay/blend-design-system'

function MyApp() {
    return (
        <div>
            <Alert
                heading="Welcome"
                description="Your Blend setup is complete!"
                variant={AlertVariant.PRIMARY}
                style={AlertStyle.SUBTLE}
            />

            <TextInput
                label="Email"
                placeholder="Enter your email"
                value=""
                onChange={(e) => console.log(e.target.value)}
            />

            <Button
                buttonType={ButtonType.PRIMARY}
                size={ButtonSize.MEDIUM}
                text="Get Started"
                onClick={() => console.log('Button clicked')}
            />
        </div>
    )
}
```

### 3. Custom Theme (Optional)

`ThemeProvider` ([source](https://github.com/juspay/blend-design-system/blob/dev/packages/blend/lib/context/ThemeProvider.tsx)) automatically loads Blend’s `FOUNDATION_THEME` and generates component tokens if you don’t pass anything. When you want to tailor the look and feel, pick the approach that fits your needs:

#### Option 1: Extend Blend foundation tokens

Use the exported `FOUNDATION_THEME` as a baseline and update only the values you need—colors, typography, spacing, and more:

```tsx
import { ThemeProvider, FOUNDATION_THEME } from '@juspay/blend-design-system'

const customFoundation = {
    ...FOUNDATION_THEME,
    colors: {
        ...FOUNDATION_THEME.colors,
        primary: {
            ...FOUNDATION_THEME.colors.primary,
            500: '#your-brand-color',
        },
    },
}

function App() {
    return (
        <ThemeProvider foundationTokens={customFoundation}>
            <YourAppContent />
        </ThemeProvider>
    )
}
```

#### Option 2: Provide a complete theme

Define your own foundation + component tokens (and optionally breakpoints) when you want 100% control over the design language. The `SidebarDemo` ([source](https://github.com/juspay/blend-design-system/blob/dev/apps/site/src/demos/SidebarDemo.tsx)) shows this pattern in action.

```tsx
import { ThemeProvider } from '@juspay/blend-design-system'
import { ALT_FOUNDATION_TOKENS } from './themes/ALT_FOUNDATION_TOKENS'
import { HDFC_COMPONENT_TOKENS } from './themes/HDFC_COMPONENT_TOKENS'
import { CUSTOM_BREAKPOINTS } from './themes/breakpoints'

function App() {
    return (
        <ThemeProvider
            foundationTokens={ALT_FOUNDATION_TOKENS}
            componentTokens={HDFC_COMPONENT_TOKENS}
            breakpoints={CUSTOM_BREAKPOINTS}
            theme="light"
        >
            <YourAppContent />
        </ThemeProvider>
    )
}
```

Use whichever option matches your project’s branding needs—start simple, then opt in to deeper customization as required. For complete examples of custom token sets, check out the theme definitions in our [GitHub repo](https://github.com/juspay/blend-design-system/blob/dev/apps/site/src/themes).

> **Component-level tokens**
>
> Every component ships with its own `ComponentNameComponentTokens` type (documented on each component page under **Component Tokens**). To adapt a component:
>
> 1. Open the component doc and review the token table—it shows the exact shape (`backgroundColor.primary.default.default`, etc.).
> 2. Import the token type so TypeScript guides you while editing.
> 3. Spread the existing token object and override just the keys you need. Values accept any valid CSS string—use foundation tokens for consistency or drop in a custom color/gradient for brand-specific styling.

```tsx
import { ThemeProvider, FOUNDATION_THEME } from '@juspay/blend-design-system'
import { ButtonComponentTokens } from '@juspay/blend-design-system/components/Button/button.tokens'
import { HDFC_COMPONENT_TOKENS } from './themes/HDFC_COMPONENT_TOKENS'

const customButtonTokens: ButtonComponentTokens = {
    ...HDFC_COMPONENT_TOKENS.button,
    backgroundColor: {
        ...HDFC_COMPONENT_TOKENS.button.backgroundColor,
        primary: {
            ...HDFC_COMPONENT_TOKENS.button.backgroundColor.primary,
            default: {
                ...HDFC_COMPONENT_TOKENS.button.backgroundColor.primary.default,
                default: '#1F2933', // direct brand color
                hover: FOUNDATION_THEME.colors.gray[900], // still reference foundation if you prefer
            },
        },
    },
}

function App() {
    return (
        <ThemeProvider
            componentTokens={{
                ...HDFC_COMPONENT_TOKENS,
                button: customButtonTokens,
            }}
        >
            <YourAppContent />
        </ThemeProvider>
    )
}
```

Any updates you make to component tokens immediately reflect in the UI—ideal for tweaking spacing, typography, states, or colors on a per-component basis while leaving the global foundation tokens untouched.

## Design Tokens

Blend includes a comprehensive set of design tokens for consistent styling:

```tsx
import { FOUNDATION_THEME } from '@juspay/blend-design-system'

// Colors
FOUNDATION_THEME.colors.primary[500] // #2B7FFF
FOUNDATION_THEME.colors.gray[100] // #F2F4F8

// Typography
FOUNDATION_THEME.fontSize.heading.lg // 24px
FOUNDATION_THEME.fontWeight[600] // 600

// Spacing
FOUNDATION_THEME.spacing[16] // 16px
FOUNDATION_THEME.borderRadius[8] // 8px
```

## Best Practices

### Component Usage

- Always refer to our [component documentation](/docs/components) for complete API reference
- Follow established patterns for consistent implementation
- Import only what you need - Blend is tree-shakeable

### Design Tokens

- Use design tokens instead of hardcoded values
- Maintain consistency with the design system guidelines
- Customize tokens through ThemeProvider for brand-specific styling

## Development Environment Setup

For contributors or those wanting to run the project locally:

### Prerequisites

- **Node.js** 18+ and pnpm
- **Git** for version control

### Setup Steps

```bash
# Clone the repository
git clone https://github.com/juspay/blend-design-system.git
cd blend-design-system

# Install dependencies
pnpm install

# Build the design system
pnpm build

# Start development mode
pnpm dev
```
