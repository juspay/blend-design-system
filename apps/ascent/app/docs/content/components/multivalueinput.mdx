## Usage

```tsx
import { MultiValueInput } from '@juspay/blend-design-system'

function MyComponent() {
    const [skills, setSkills] = useState(['JavaScript', 'React'])

    const handleTagAdd = (tag: string) => {
        setSkills((prev) => [...prev, tag])
    }

    const handleTagRemove = (tagToRemove: string) => {
        setSkills((prev) => prev.filter((tag) => tag !== tagToRemove))
    }

    return (
        <MultiValueInput
            label="Technical Skills"
            placeholder="Type a skill and press Enter"
            tags={skills}
            onTagAdd={handleTagAdd}
            onTagRemove={handleTagRemove}
            hintText="Add your technical skills"
        />
    )
}
```

## Features

- **Tag-Based Input**: Visual representation of multiple values as removable tags
- **Keyboard Interactions**: Enter to add, Backspace to remove last tag
- **Duplicate Prevention**: Automatically prevents adding duplicate values
- **Interactive Tags**: Click X icon or use backspace to remove individual tags
- **Flexible Container**: Responsive wrapping layout that adapts to content
- **Validation States**: Built-in error styling with custom error messages
- **Focus Management**: Automatic focus handling for seamless user experience
- **Accessibility**: Full keyboard navigation and screen reader support
- **Form Integration**: Native HTML input attributes and form compatibility

## Usage Examples

### Skills Input

Multi-value input for collecting technical skills:

```tsx
import { MultiValueInput } from '@juspay/blend-design-system'

const [skills, setSkills] = useState(['JavaScript', 'TypeScript', 'React'])

const handleAddSkill = (skill: string) => {
    setSkills((prev) => [...prev, skill])
}

const handleRemoveSkill = (skillToRemove: string) => {
    setSkills((prev) => prev.filter((skill) => skill !== skillToRemove))
}

;<MultiValueInput
    label="Technical Skills"
    placeholder="Type a skill and press Enter"
    tags={skills}
    onTagAdd={handleAddSkill}
    onTagRemove={handleRemoveSkill}
    hintText="Add technologies you work with"
    required
/>
```

### Email Recipients

Input for multiple email addresses:

```tsx
const [recipients, setRecipients] = useState(['john@example.com'])

const validateEmail = (email: string) => {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)
}

const handleAddEmail = (email: string) => {
    if (validateEmail(email)) {
        setRecipients((prev) => [...prev, email])
    }
}

const handleRemoveEmail = (emailToRemove: string) => {
    setRecipients((prev) => prev.filter((email) => email !== emailToRemove))
}

;<MultiValueInput
    label="Email Recipients"
    placeholder="Enter email address and press Enter"
    tags={recipients}
    onTagAdd={handleAddEmail}
    onTagRemove={handleRemoveEmail}
    hintText="Add email addresses for recipients"
    error={recipients.some((email) => !validateEmail(email))}
    errorMessage="Please enter valid email addresses"
/>
```

### Keywords/Tags

Input for content keywords or tags:

```tsx
const [keywords, setKeywords] = useState(['design', 'ui', 'ux'])

const handleAddKeyword = (keyword: string) => {
    // Convert to lowercase and limit length
    const processedKeyword = keyword.toLowerCase().slice(0, 20)
    if (processedKeyword.length >= 2) {
        setKeywords((prev) => [...prev, processedKeyword])
    }
}

const handleRemoveKeyword = (keywordToRemove: string) => {
    setKeywords((prev) => prev.filter((keyword) => keyword !== keywordToRemove))
}

;<MultiValueInput
    label="Tags"
    placeholder="Add tags (min 2 characters)"
    tags={keywords}
    onTagAdd={handleAddKeyword}
    onTagRemove={handleRemoveKeyword}
    hintText="Add relevant tags for your content"
    size="small"
/>
```

### Programming Languages

Input with validation and custom processing:

```tsx
const [languages, setLanguages] = useState(['JavaScript', 'Python'])

const popularLanguages = [
    'JavaScript',
    'Python',
    'Java',
    'TypeScript',
    'C++',
    'C#',
    'PHP',
    'Ruby',
    'Go',
    'Rust',
]

const handleAddLanguage = (language: string) => {
    // Capitalize first letter
    const formattedLanguage =
        language.charAt(0).toUpperCase() + language.slice(1).toLowerCase()
    setLanguages((prev) => [...prev, formattedLanguage])
}

const handleRemoveLanguage = (languageToRemove: string) => {
    setLanguages((prev) => prev.filter((lang) => lang !== languageToRemove))
}

;<MultiValueInput
    label="Programming Languages"
    sublabel="Languages you're proficient in"
    placeholder="Enter programming language"
    tags={languages}
    onTagAdd={handleAddLanguage}
    onTagRemove={handleRemoveLanguage}
    hintText={`Popular: ${popularLanguages.slice(0, 3).join(', ')}, etc.`}
    size="large"
/>
```

## Behavior

### Keyboard Interactions

- **Enter Key**: Adds the current input value as a new tag
- **Backspace Key**: When input is empty, removes the last tag
- **Tab Navigation**: Standard tab order through tags and input
- **Focus Management**: Clicking container focuses the input field

### Tag Management

- **Duplicate Prevention**: Automatically prevents adding existing tags
- **Trim Whitespace**: Input values are automatically trimmed
- **Empty Value Handling**: Empty or whitespace-only inputs are ignored
- **Case Sensitivity**: Tag comparison is case-sensitive by default

### Visual Feedback

- **Focus States**: Container shows focus styling when input is active
- **Hover Effects**: Visual feedback on hover for better interactivity
- **Error States**: Unified error styling for validation feedback
- **Tag Removal**: Clear visual indicators for tag removal actions

### Responsive Layout

- **Flex Wrapping**: Tags automatically wrap to new lines as needed
- **Dynamic Sizing**: Container adjusts height based on content
- **Consistent Spacing**: Uniform gaps between tags and input field
- **Size Adaptation**: Padding and spacing adapt to size variant

## Best Practices

### When to Use MultiValueInput

- **Multiple Keywords**: Tags, categories, or keywords for content
- **Skills/Technologies**: Technical skills, programming languages, tools
- **Contact Lists**: Email addresses, usernames, or contact identifiers
- **Filtering**: Multiple filter criteria or search terms
- **Configuration**: Multiple values for settings or preferences

### Input Validation

- **Format Validation**: Validate input format before adding tags
- **Length Limits**: Implement reasonable limits for individual tags and total count
- **Custom Processing**: Transform input values (capitalize, format, etc.)
- **Duplicate Handling**: Decide whether to allow or prevent duplicates
- **Error Feedback**: Provide clear validation messages

### User Experience

- **Clear Instructions**: Use hint text to explain interaction patterns
- **Visual Cues**: Make tag removal actions clearly visible
- **Keyboard Shortcuts**: Ensure Enter and Backspace work as expected
- **Placeholder Text**: Provide helpful placeholder text for context
- **Size Selection**: Choose appropriate size for form context

### Data Management

- **State Structure**: Use arrays for simple strings, objects for complex data
- **Persistence**: Save tag state appropriately for form submission
- **Performance**: Consider virtualization for very large tag lists
- **Accessibility**: Ensure screen readers can understand tag structure

### Accessibility Considerations

- **Label Association**: Proper labeling for screen readers
- **Tag Announcement**: Ensure tags are announced when added/removed
- **Keyboard Navigation**: Full keyboard accessibility for all interactions
- **Focus Indicators**: Visible focus indicators for keyboard users
- **Error Communication**: Clear error messages for validation failures

### Integration Patterns

- **Form Libraries**: Compatible with React Hook Form, Formik, etc.
- **Validation**: Integrate with form validation libraries
- **Autocomplete**: Combine with suggestion systems for better UX
- **Data Sources**: Connect to APIs or static data for validation/formatting
- **State Management**: Use with Redux, Zustand, or other state libraries
