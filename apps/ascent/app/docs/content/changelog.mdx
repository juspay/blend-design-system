# Changelog

Stay up-to-date with all changes, improvements, and new features in the Blend Design System. This page contains a comprehensive log of all releases and updates.

## About Our Versioning

The Blend Design System follows [Semantic Versioning](https://semver.org/):

- **Major (X.0.0)**: Breaking changes that require migration
- **Minor (0.X.0)**: New features, components, or enhancements
- **Patch (0.0.X)**: Bug fixes, documentation improvements, and small updates

## Change Categories

We categorize our changes to help you understand what's new:

- 🎨 **New Components**: Brand new components added to the design system
- ✨ **Component Enhancements**: New props, variants, or functionality for existing components
- 🐛 **Bug Fixes**: Component behavior fixes and corrections
- 💥 **Breaking Changes**: API changes that require migration
- 🔧 **Design Tokens**: Updates to colors, spacing, typography, or other design tokens
- 📚 **Documentation**: Documentation improvements and updates
- 🔒 **Accessibility**: Accessibility improvements and fixes
- ⚡ **Performance**: Component optimization and performance improvements

## Recent Releases

### v0.0.5 - Current Version

The current version includes a comprehensive set of components with the following features:

#### Components Available

- **Form Controls**: Button, ButtonGroup, Checkbox, Radio, Switch, SingleSelect, MultiSelect
- **Layout**: Accordion, Modal, Popover, Tooltip, Tabs, Sidebar
- **Data Display**: Alert, Avatar, AvatarGroup, Breadcrumb, Tags, SplitTag, StatCard, Charts
- **Input Components**: TextInput, TextArea, SearchInput, NumberInput, OTPInput, UnitInput, MultiValueInput, DropdownInput
- **Navigation**: Menu, Breadcrumb, Sidebar
- **Feedback**: Snackbar, Alert, Modal
- **Data Visualization**: Charts, StatCard, DataTable

#### Design Tokens

- Complete color system with light and dark mode support
- Typography scale with responsive design
- Spacing and layout tokens
- Border radius and shadow tokens
- Opacity and transition tokens

#### Features

- **Accessibility**: WCAG AA compliant components with proper ARIA support
- **Responsive Design**: Mobile-first approach with responsive breakpoints
- **Theme Support**: Light and dark mode with custom theme configuration
- **TypeScript**: Full TypeScript support with comprehensive type definitions
- **Testing**: Comprehensive test coverage with accessibility testing
- **Storybook**: Interactive component documentation and playground
