## Usage

```tsx
import {
    UnitInput,
    UnitInputSize,
    UnitPosition,
} from '@juspay/blend-design-system'

function MyComponent() {
    const [weight, setWeight] = useState<number | undefined>(undefined)

    const handleWeightChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        setWeight(e.target.value ? parseFloat(e.target.value) : undefined)
    }

    return (
        <UnitInput
            label="Weight"
            placeholder="Enter weight"
            value={weight}
            onChange={handleWeightChange}
            unit="kg"
            unitPosition={UnitPosition.RIGHT}
            size={UnitInputSize.MEDIUM}
            min={0}
            max={1000}
            step={0.1}
            hintText="Enter weight in kilograms"
        />
    )
}
```

## Features

- **Integrated Unit Display**: Built-in unit label with configurable positioning
- **Multiple Sizes**: Small, medium, and large size variants with responsive behavior
- **Numeric Input**: Number input type with min, max, and step validation
- **Slot Support**: Configurable left and right slots for icons or additional elements
- **Responsive Labels**: Floating labels on small screens for large size variant
- **Error States**: Visual error feedback with custom error messages
- **Form Integration**: Full form library support with validation
- **Accessibility**: Screen reader support and proper focus management
- **Dynamic Padding**: Automatically adjusts padding based on unit and slot positioning
- **Default Icon**: Includes Weight icon as default right slot

## Behavior

### Unit Positioning

- **Right Position**: Unit appears on the right side with gray background
- **Left Position**: Unit appears on the left side with gray background
- **Border Integration**: Unit has matching border styling with the input
- **Dynamic Width**: Unit container adjusts width based on content length

### Size Variants

- **Medium Size**: Standard padding and font size for most use cases
- **Large Size**: Increased padding with floating labels on small screens
- **Responsive Behavior**: Large size shows floating labels on mobile devices
- **Consistent Spacing**: Maintains proper spacing regardless of size

### Slot Management

- **Left Slot**: Positioned between left edge and input, adjusted for unit position
- **Right Slot**: Positioned between input and right edge, includes default Weight icon
- **Dynamic Padding**: Input padding automatically adjusts based on slot widths
- **Icon Integration**: Slots integrate seamlessly with unit positioning

### Focus and Validation

- **Focus States**: Visual focus indicators with border and shadow changes
- **Numeric Validation**: Enforces numeric input with min, max, and step constraints
- **Error Feedback**: Red borders and error message display for invalid states
- **Required Fields**: Visual indicators for required form fields

## Best Practices

### When to Use UnitInput

- **Measurements**: Weight, height, distance, temperature, speed
- **Quantities**: Inventory counts, limits, quotas with units
- **Financial**: Prices, amounts, budgets with currency units
- **Scientific**: Laboratory measurements, calculations with units
- **Configuration**: Settings that require specific units or measurements

### Unit Selection Guidelines

- **Clear Units**: Use widely recognized unit abbreviations (kg, cm, $, %)
- **Consistent Units**: Maintain unit consistency within related inputs
- **Localization**: Consider locale-specific units and formats
- **Abbreviations**: Keep unit text short to maintain clean appearance
- **Context**: Choose units appropriate for the target audience

### Size and Layout

- **Medium Size**: Default choice for most form layouts
- **Large Size**: Use for prominent inputs or mobile-first designs
- **Responsive Design**: Test floating label behavior on small screens
- **Form Consistency**: Maintain consistent sizing within form groups
- **Visual Hierarchy**: Use size to indicate input importance

### Accessibility Considerations

- **Label Association**: Ensure proper labeling including unit information
- **Screen Readers**: Unit information should be accessible to assistive technology
- **Focus Management**: Maintain clear focus indicators and keyboard navigation
- **Error Communication**: Make validation errors accessible and specific
- **Help Text**: Provide clear instructions about expected values and units

### Validation and Error Handling

- **Range Validation**: Set appropriate min/max values for the context
- **Step Validation**: Use logical step values for the measurement type
- **Unit Conversion**: Consider providing unit conversion helpers
- **Error Messages**: Provide specific, actionable error messages
- **Real-time Validation**: Validate input as user types when appropriate

### Form Integration

- **Controlled Components**: Always use controlled inputs with proper state management
- **Data Types**: Handle number/undefined types appropriately
- **Submission**: Convert and validate units before form submission
- **Default Values**: Provide sensible defaults when appropriate
- **Reset Functionality**: Implement proper form reset behavior

### Performance Optimization

- **State Management**: Use appropriate state patterns for complex forms
- **Debouncing**: Consider debouncing for expensive validation operations
- **Memory Management**: Clean up event listeners and effects properly
- **Render Optimization**: Memoize expensive calculations and validations

### User Experience

- **Clear Placeholders**: Use descriptive placeholder text
- **Helpful Hints**: Provide context about expected ranges and formats
- **Progressive Enhancement**: Start with basic functionality
- **Visual Feedback**: Show clear confirmation of successful input
- **Error Recovery**: Allow easy correction of invalid inputs
