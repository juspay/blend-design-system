## Usage

```tsx
import { OTPInput } from '@juspay/blend-design-system'

function MyComponent() {
    const [otp, setOtp] = useState('')

    const handleOTPChange = (value: string) => {
        setOtp(value)
        if (value.length === 6) {
            // Handle complete OTP
            console.log('Complete OTP:', value)
        }
    }

    return (
        <OTPInput
            label="Verification Code"
            value={otp}
            onChange={handleOTPChange}
            length={6}
            autoFocus
            hintText="Enter the 6-digit code sent to your phone"
        />
    )
}
```

## Features

- **Individual Digit Inputs**: Separate input field for each OTP digit
- **Auto-Focus Management**: Automatic focus progression and regression
- **Keyboard Navigation**: Arrow key navigation between inputs
- **Clipboard Support**: Smart paste functionality with digit extraction
- **Numeric Validation**: Only accepts numeric digits (0-9)
- **Customizable Length**: Configurable number of digits (typically 4-8)
- **Error States**: Visual error feedback for invalid codes
- **Accessibility**: Full keyboard navigation and screen reader support
- **Form Integration**: Compatible with form libraries and validation
- **Auto-Focus**: Optional auto-focus on component mount

## Usage Examples

### Basic OTP Input

Simple 6-digit OTP input for verification:

```tsx
import { OTPInput } from '@juspay/blend-design-system'

const [verificationCode, setVerificationCode] = useState('')

const handleVerificationChange = (value: string) => {
    setVerificationCode(value)

    if (value.length === 6) {
        // Auto-submit when complete
        verifyCode(value)
    }
}

;<OTPInput
    label="Verification Code"
    value={verificationCode}
    onChange={handleVerificationChange}
    length={6}
    autoFocus
    hintText="Enter the 6-digit code sent to your email"
/>
```

### SMS Authentication

4-digit SMS OTP with custom validation:

```tsx
const [smsCode, setSmsCode] = useState('')
const [isInvalid, setIsInvalid] = useState(false)

const handleSMSCodeChange = (value: string) => {
    setSmsCode(value)
    setIsInvalid(false) // Clear error on new input

    if (value.length === 4) {
        validateSMSCode(value)
    }
}

const validateSMSCode = async (code: string) => {
    try {
        await verifySMSCode(code)
        // Handle success
    } catch (error) {
        setIsInvalid(true)
    }
}

;<OTPInput
    label="SMS Code"
    sublabel="Verify your phone number"
    value={smsCode}
    onChange={handleSMSCodeChange}
    length={4}
    error={isInvalid}
    errorMessage="Invalid code. Please try again."
    hintText="Enter the 4-digit code sent via SMS"
/>
```

### Email Verification

6-digit email verification with resend functionality:

```tsx
const [emailCode, setEmailCode] = useState('')
const [timeLeft, setTimeLeft] = useState(300) // 5 minutes
const [canResend, setCanResend] = useState(false)

const handleEmailCodeChange = (value: string) => {
    setEmailCode(value)
}

const resendCode = () => {
    setEmailCode('')
    setTimeLeft(300)
    setCanResend(false)
    // Resend email verification code
}

;<div>
    <OTPInput
        label="Email Verification"
        value={emailCode}
        onChange={handleEmailCodeChange}
        length={6}
        hintText={`Enter the code sent to your email ${canResend ? '' : `(${Math.floor(timeLeft / 60)}:${(timeLeft % 60).toString().padStart(2, '0')})`}`}
        required
    />

    {canResend && (
        <button onClick={resendCode} className="mt-2 text-blue-600">
            Resend Code
        </button>
    )}
</div>
```

### Custom Length OTP

Variable length OTP based on security requirements:

```tsx
const [customOTP, setCustomOTP] = useState('')
const [otpLength, setOtpLength] = useState(6)

const handleCustomOTPChange = (value: string) => {
    setCustomOTP(value)

    if (value.length === otpLength) {
        console.log(`Complete ${otpLength}-digit OTP:`, value)
    }
}

;<div>
    <div className="mb-4">
        <label>OTP Length:</label>
        <select
            value={otpLength}
            onChange={(e) => {
                setOtpLength(Number(e.target.value))
                setCustomOTP('') // Clear on length change
            }}
        >
            <option value={4}>4 digits</option>
            <option value={6}>6 digits</option>
            <option value={8}>8 digits</option>
        </select>
    </div>

    <OTPInput
        label="Custom Length OTP"
        value={customOTP}
        onChange={handleCustomOTPChange}
        length={otpLength}
        hintText={`Enter ${otpLength}-digit verification code`}
    />
</div>
```

## Behavior

### Focus Management

- **Auto-Focus**: Optionally focuses first input on mount
- **Forward Focus**: Automatically moves to next input when digit is entered
- **Backward Focus**: Moves to previous input on backspace when current is empty
- **Arrow Navigation**: Left/right arrow keys move between inputs
- **Click Focus**: Clicking any input focuses that specific field

### Input Handling

- **Numeric Only**: Accepts only digits (0-9), ignores other characters
- **Single Digit**: Each input accepts only one digit, replacing existing
- **Backspace Logic**: Clears current digit or moves to previous input
- **Overflow Handling**: Extra characters are ignored, maintains single digit per input

### Clipboard Support

- **Smart Paste**: Paste operation on first input distributes digits across all inputs
- **Digit Extraction**: Automatically extracts numeric digits from pasted content
- **Length Limiting**: Only uses digits up to the specified length
- **Focus Positioning**: Focuses appropriate input after paste operation

### State Synchronization

- **Value Binding**: Component value stays synchronized with parent state
- **Change Callbacks**: Calls onChange for every digit modification
- **Error Propagation**: Error state applies to all input fields
- **Disabled State**: Disabling affects all inputs uniformly

## Best Practices

### When to Use OTPInput

- **Email Verification**: Confirm email addresses during registration
- **SMS Authentication**: Phone number verification workflows
- **Two-Factor Authentication**: Secondary authentication step
- **Password Reset**: Secure password reset verification
- **Transaction Confirmation**: High-security transaction verification

### Security Considerations

- **Time Limits**: Implement appropriate code expiration times
- **Rate Limiting**: Prevent brute force attacks with rate limiting
- **Secure Transmission**: Always use HTTPS for OTP transmission
- **Auto-Clear**: Clear codes after successful verification
- **Attempt Limits**: Limit the number of verification attempts

### User Experience

- **Clear Instructions**: Provide clear context about where the code is sent
- **Visual Feedback**: Show loading states during verification
- **Error Handling**: Provide specific, actionable error messages
- **Resend Options**: Offer code resend functionality with appropriate delays
- **Auto-Submit**: Consider auto-submitting when code is complete

### Length Configuration

- **4 Digits**: Quick SMS verification, lower security scenarios
- **6 Digits**: Standard email verification, moderate security
- **8 Digits**: High-security scenarios, sensitive operations
- **Custom Length**: Match your backend OTP generation settings

### Accessibility Considerations

- **Label Association**: Ensure proper labeling for screen readers
- **Input Announcement**: Verify each input is properly announced
- **Keyboard Navigation**: Test all keyboard interactions thoroughly
- **Focus Indicators**: Maintain visible focus indicators
- **Error Communication**: Ensure error messages are accessible

### Form Integration

- **Controlled Components**: Always use controlled inputs with proper state
- **Validation**: Integrate with form validation libraries
- **Submission**: Handle form submission when OTP is complete
- **Reset Functionality**: Implement proper form reset behavior
- **Loading States**: Show appropriate loading states during verification

### Performance Optimization

- **Debouncing**: Consider debouncing for expensive validation operations
- **Memoization**: Use React.memo for input optimization if needed
- **State Management**: Use appropriate state management patterns
- **Memory Cleanup**: Properly clean up timers and event listeners

### Error Handling

- **Validation Timing**: Validate only when OTP is complete
- **Clear Messaging**: Provide specific error messages for different scenarios
- **Recovery**: Allow easy recovery from error states
- **Visual Feedback**: Use consistent error styling across all inputs
