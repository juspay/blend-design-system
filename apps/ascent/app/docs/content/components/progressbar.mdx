## Usage

```tsx
import {
    ProgressBar,
    ProgressBarVariant,
    ProgressBarSize,
    ProgressBarType,
} from 'blend-v1'
import { useState } from 'react'

function MyComponent() {
    const [progress, setProgress] = useState(65)

    return (
        <div className="space-y-4">
            <ProgressBar
                value={progress}
                variant={ProgressBarVariant.SOLID}
                size={ProgressBarSize.MEDIUM}
                showLabel={true}
            />

            <ProgressBar
                value={progress}
                variant={ProgressBarVariant.CIRCULAR}
                type={ProgressBarType.SOLID}
                size={ProgressBarSize.MEDIUM}
                showLabel={true}
            />
        </div>
    )
}
```

## Features

- **Multiple Variants**: Linear solid, linear segmented, and circular progress indicators
- **Size Options**: Three size variants (small, medium, large) for different contexts
- **Progress Labels**: Optional percentage labels with consistent typography
- **Smooth Animations**: Built-in transitions for smooth progress updates
- **Value Clamping**: Automatic clamping of values between 0-100 range
- **Circular Indicators**: Full circular progress with solid and segmented styles
- **Segmented Styling**: Distinctive segmented appearance for step-based progress
- **Responsive Design**: Adapts to container width for linear variants
- **Accessibility Ready**: Semantic structure suitable for ARIA enhancements
- **Token-Based Styling**: Comprehensive design token system for customization
- **High Performance**: Optimized rendering for smooth animations
- **Type Safety**: Full TypeScript support with proper type definitions

## Behavior

### Value Management

- **Range Clamping**: Values are automatically clamped between 0 and 100
- **Real-time Updates**: Progress updates smoothly with CSS transitions
- **Percentage Display**: Labels show rounded percentage values
- **Responsive Width**: Linear variants adapt to container width

### Visual Feedback

- **Smooth Transitions**: 0.3s ease-in-out transitions for progress changes
- **Clear Indication**: Visual distinction between completed and remaining progress
- **Consistent Sizing**: Proportional scaling across size variants
- **Label Positioning**: Appropriately positioned labels for readability

### Animation System

- **CSS Transitions**: Hardware-accelerated transitions for smooth performance
- **Stroke Animation**: Circular variants animate stroke-dashoffset for smooth circular progress
- **Width Animation**: Linear variants animate width property for smooth filling
- **No Flicker**: Optimized rendering to prevent visual artifacts

## Best Practices

### When to Use Progress Bars

- **File Operations**: For file uploads, downloads, or processing
- **Multi-step Processes**: For wizards, forms, or setup procedures
- **Loading States**: For data loading or content generation
- **Background Tasks**: For long-running operations with known duration

### Progress Bar Guidelines

- **Accurate Values**: Ensure progress values accurately reflect actual completion
- **Appropriate Variants**: Use segmented for step-based processes, solid for continuous progress
- **Clear Context**: Provide descriptive labels explaining what's progressing
- **Size Selection**: Choose sizes appropriate for the UI context and importance

### Visual Design

- **Consistent Sizing**: Use consistent progress bar sizes throughout your application
- **Proper Spacing**: Ensure adequate spacing around progress bars
- **Label Visibility**: Show labels when progress percentage is important to users
- **Color Contrast**: Ensure sufficient contrast between progress and background

### Performance Considerations

- **Smooth Updates**: Avoid frequent rapid updates that could cause visual jitter
- **Animation Optimization**: Leverage CSS transitions for smooth visual feedback
- **Value Throttling**: Consider throttling progress updates for better performance
- **Memory Management**: Clean up intervals or timeouts used for progress simulation

### Accessibility Considerations

- **ARIA Labels**: Consider adding ARIA labels for screen readers
- **Progress Context**: Provide context about what process is being tracked
- **Completion Feedback**: Ensure users are notified when progress completes
- **Alternative Text**: Provide alternative indicators for users who can't see progress bars

### Mobile Considerations

- **Touch-Friendly Sizing**: Ensure progress bars are visible on mobile screens
- **Performance Optimization**: Optimize animations for mobile devices
- **Battery Efficiency**: Consider reducing animation frequency on mobile
- **Screen Real Estate**: Choose appropriate sizes for mobile layouts
