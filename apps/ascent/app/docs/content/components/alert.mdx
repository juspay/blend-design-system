## Usage

```tsx
import { Alert, AlertVariant, AlertStyle, AlertActionPlacement } from 'blend-v1'

function MyComponent() {
    return (
        <Alert
            heading="Update Available"
            description="A new version of the application is available. Please update to get the latest features."
            variant={AlertVariant.PRIMARY}
            style={AlertStyle.SUBTLE}
            primaryAction={{
                label: 'Update Now',
                onClick: () => console.log('Update clicked'),
            }}
            secondaryAction={{
                label: 'Later',
                onClick: () => console.log('Later clicked'),
            }}
            onClose={() => console.log('Alert dismissed')}
        />
    )
}
```

## Features

- **7 Semantic Variants**: Primary, Success, Warning, Error, Purple, Orange, and Neutral for different message types
- **2 Visual Styles**: Subtle (filled background) and No Fill (outline only) for different design contexts
- **Flexible Actions**: Support for primary and secondary action buttons with customizable placement
- **Dismissible**: Optional close button with callback functionality
- **Custom Icons**: Support for custom icons to enhance visual communication
- **Responsive Layout**: Automatic layout adjustment based on content and screen size
- **Accessible Design**: Full keyboard navigation and screen reader support
- **Action Placement**: Buttons can be positioned on the right or bottom for optimal layout
- **Max Width Control**: Built-in max-width of 900px for optimal readability
- **Theme Integration**: Consistent styling through design tokens

## Usage Examples

### Basic Information Alert

Simple alert with heading and description only

```tsx

    heading="Information"
    description="This is an important informational message for the user."
```

Alert confirming successful action with dismissible option

```tsx

    heading="Success!"
    description="Your changes have been saved successfully."
```

### Warning Alert with Actions

Alert requiring user decision with action buttons

## Behavior

### Action Placement Logic

- **Right Placement**: When both primary and secondary actions are provided
- **Bottom Placement**: When `actionPlacement={AlertActionPlacement.BOTTOM}` is explicitly set
- **Automatic Bottom**: When only `onClose` is provided without other actions

### Close Button Behavior

- **Auto-positioning**: Close button automatically moves to bottom when no other actions are present
- **Always Visible**: Close button remains accessible when `onClose` prop is provided
- **Icon Consistency**: Uses standard X icon from Lucide React

### Responsive Behavior

- **Max Width**: Component has a maximum width of 900px for optimal readability
- **Flexible Layout**: Content and actions adapt to available space
- **Mobile Friendly**: Action buttons stack appropriately on smaller screens

## Accessibility

- **Semantic HTML**: Uses proper ARIA roles and landmarks for alert content
- **Keyboard Navigation**: All interactive elements are keyboard accessible
- **Screen Reader Support**: Proper announcements for alert content and state changes
- **Focus Management**: Clear focus indicators on all interactive elements
- **Color Independence**: Alert meaning conveyed through text, not color alone
- **High Contrast**: Supports high contrast mode and custom color schemes

```tsx
    PRIMARY = 'primary', // Blue color scheme for general information

    PURPLE = 'purple', // Purple color scheme for feature announcements
    ORANGE = 'orange', // Orange color scheme for marketing messages
```
