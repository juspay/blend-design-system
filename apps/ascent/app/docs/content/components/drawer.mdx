## Usage

```tsx
import {
    Drawer,
    DrawerTrigger,
    DrawerPortal,
    DrawerOverlay,
    DrawerContent,
    DrawerHeader,
    DrawerTitle,
    DrawerDescription,
    DrawerBody,
    DrawerFooter,
    DrawerClose,
    Button,
    ButtonType,
} from 'blend-v1'
import { Menu } from 'lucide-react'
import { useState } from 'react'

function MyComponent() {
    const [isOpen, setIsOpen] = useState(false)

    return (
        <Drawer open={isOpen} onOpenChange={setIsOpen}>
            <DrawerTrigger>
                <Button
                    text="Open Menu"
                    buttonType={ButtonType.SECONDARY}
                    leadingIcon={<Menu size={16} />}
                />
            </DrawerTrigger>

            <DrawerPortal>
                <DrawerOverlay />
                <DrawerContent direction="left">
                    <DrawerHeader>
                        <DrawerTitle>Navigation Menu</DrawerTitle>
                        <DrawerDescription>
                            Choose from the options below to navigate
                        </DrawerDescription>
                    </DrawerHeader>

                    <DrawerBody>
                        <div className="space-y-4">
                            <button className="w-full text-left p-3 hover:bg-gray-100 rounded">
                                Dashboard
                            </button>
                            <button className="w-full text-left p-3 hover:bg-gray-100 rounded">
                                Projects
                            </button>
                            <button className="w-full text-left p-3 hover:bg-gray-100 rounded">
                                Settings
                            </button>
                        </div>
                    </DrawerBody>

                    <DrawerFooter>
                        <DrawerClose>
                            <Button
                                text="Close"
                                buttonType={ButtonType.SECONDARY}
                                fullWidth
                            />
                        </DrawerClose>
                    </DrawerFooter>
                </DrawerContent>
            </DrawerPortal>
        </Drawer>
    )
}
```

## Features

- **Full-Screen Overlay**: Uses DrawerPortal and DrawerOverlay to render above all app content with proper z-index stacking
- **Multi-Directional**: Slide from any side (top, bottom, left, right) with responsive positioning
- **Modular Components**: Composable header, body, and footer components for flexible layouts
- **Snap Points**: Support for multiple snap positions on bottom/top drawers
- **Drag Handle**: Optional drag handle for bottom/top drawers with gesture support
- **Mobile Optimized**: Responsive design with mobile-specific positioning and offsets
- **Overlay Control**: Optional modal overlay with backdrop click dismissal
- **Nested Support**: Ability to stack drawers for complex navigation flows
- **Built on Vaul**: Leverages Vaul library for robust gesture and animation support
- **Custom Styling**: Comprehensive token system for complete visual customization
- **Accessibility First**: Full keyboard navigation, focus management, and screen reader support
- **Content-Driven Height**: Automatic sizing based on content with overflow handling
- **Sequential Snapping**: Option to disable velocity-based snapping for controlled navigation

## Behavior

### Slide Animation

- **Direction Control**: Drawers slide in from the specified edge (top, bottom, left, right)
- **Smooth Transitions**: Built-in spring animations for opening and closing
- **Gesture Support**: Drag to open/close with velocity-based snapping
- **Snap Points**: Multiple stopping positions for flexible sizing

### State Management

- **Controlled Mode**: Use `open` and `onOpenChange` props for external state control
- **Uncontrolled Mode**: Component manages its own open/closed state internally
- **Nested Support**: Multiple drawers can be stacked for complex navigation

### Mobile Behavior

- **Responsive Positioning**: Custom mobile offsets for optimal mobile experience
- **Touch Gestures**: Full gesture support for drag-to-open and drag-to-close
- **Safe Areas**: Respects mobile safe areas and notches
- **Content Adaptation**: Automatic sizing adjustments for mobile viewports

### Content Management

- **Flexible Layout**: Modular header, body, and footer components
- **Scrollable Content**: Body content scrolls when exceeding drawer height
- **Overflow Control**: Configurable overflow behavior for different content types
- **Dynamic Sizing**: Content-driven height with maximum constraints

## Accessibility

- **Focus Management**: Focus is trapped within drawer when open and restored on close
- **Keyboard Navigation**: Full keyboard support with Tab, Shift+Tab, and Escape keys
- **Screen Reader Support**: Proper ARIA roles, labels, and live regions
- **ARIA Compliance**: Uses appropriate dialog semantics and state announcements
- **High Contrast**: Supports high contrast mode and custom color schemes
- **Motion Preferences**: Respects user's reduced motion preferences for animations

## Best Practices

### When to Use Drawers

- **Navigation Menus**: For slide-out navigation that doesn't interrupt main content
- **Detail Views**: For displaying additional information without page navigation
- **Form Dialogs**: For forms that need more space than traditional modals
- **Filter Panels**: For advanced filtering options that need persistent visibility

### Drawer Design Guidelines

- **Always Use Portal**: Always wrap DrawerContent with DrawerPortal and include DrawerOverlay for proper full-screen rendering
- **Clear Purpose**: Each drawer should have a focused, single purpose
- **Appropriate Direction**: Choose slide direction based on content type and screen size
- **Consistent Behavior**: Use consistent slide directions throughout your application
- **Content Organization**: Structure content with clear headers and logical sections

### Content Guidelines

- **Scannable Layout**: Organize content with clear hierarchy and spacing
- **Action Placement**: Place primary actions in footer for easy access
- **Scroll Indicators**: Provide visual cues when content extends beyond visible area
- **Loading States**: Show loading states for dynamic content

### Mobile Considerations

- **Touch Targets**: Ensure interactive elements meet minimum touch target sizes
- **Gesture Conflicts**: Avoid gesture conflicts with system navigation
- **Safe Areas**: Consider mobile safe areas and keyboard interactions
- **Performance**: Optimize animations and content loading for mobile devices
