## Usage

```tsx
import { Checkbox, CheckboxSize } from 'blend-v1'
import { useState } from 'react'

function MyComponent() {
    const [checked, setChecked] = useState(false)

    return (
        <Checkbox
            checked={checked}
            onCheckedChange={setChecked}
            size={CheckboxSize.MEDIUM}
            required={true}
        >
            I agree to the terms and conditions
        </Checkbox>
    )
}
```

## Features

- **3 Checkbox States**: Checked, unchecked, and indeterminate states for complex scenarios
- **2 Size Options**: Small and medium sizes for different interface contexts
- **Interactive States**: Hover, disabled, and error states with visual feedback
- **Controlled/Uncontrolled**: Support for both controlled and uncontrolled component patterns
- **Form Integration**: Seamless integration with forms including validation support
- **Accessibility First**: Full ARIA compliance and keyboard navigation support
- **Custom Content**: Support for labels, subtext, and additional content slots
- **Responsive Design**: Consistent sizing and spacing across different screen sizes
- **Error Handling**: Built-in error state styling for form validation
- **Icon Integration**: Uses Lucide React icons for consistent check and indeterminate indicators
- **Theme Integration**: Fully styled using design tokens for consistent theming

## Usage Examples

### Basic Checkbox

Simple checkbox with label text

```tsx
const [accepted, setAccepted] = useState(false)

<Checkbox
    checked={accepted}
    onCheckedChange={setAccepted}
>
    I accept the terms of service
</Checkbox>
```

### Checkbox with Subtext

Checkbox with additional descriptive text

```tsx
const [notifications, setNotifications] = useState(true)

<Checkbox
    checked={notifications}
    onCheckedChange={setNotifications}
    subtext="Receive email notifications about account activity and updates"
>
    Enable notifications
</Checkbox>
```

### Small Size Checkbox

Compact checkbox for dense layouts

```tsx
const [remember, setRemember] = useState(false)

<Checkbox
    checked={remember}
    onCheckedChange={setRemember}
    size={CheckboxSize.SMALL}
>
    Remember me
</Checkbox>
```

### Indeterminate Checkbox

Checkbox in indeterminate state (partially selected)

```tsx
const [selectAll, setSelectAll] = useState<boolean | 'indeterminate'>('indeterminate')

<Checkbox
    checked={selectAll}
    onCheckedChange={setSelectAll}
>
    Select all items
</Checkbox>
```

### Disabled Checkbox

Non-interactive checkbox in disabled state

```tsx
<Checkbox checked={true} disabled={true}>
    This option is disabled
</Checkbox>
```

### Required Checkbox

Checkbox marked as required for form validation

```tsx
const [terms, setTerms] = useState(false)

<Checkbox
    checked={terms}
    onCheckedChange={setTerms}
    required={true}
>
    I agree to the terms and conditions *
</Checkbox>
```

### Error State Checkbox

Checkbox in error state for validation feedback

```tsx
const [privacy, setPrivacy] = useState(false)

<Checkbox
    checked={privacy}
    onCheckedChange={setPrivacy}
    error={!privacy}
    required={true}
>
    I accept the privacy policy
</Checkbox>
```

### Checkbox with Custom Slot

Checkbox with additional content in the slot

### Uncontrolled Checkbox

Checkbox with default state (uncontrolled)

```tsx
<Checkbox
    defaultChecked={true}

</Checkbox>
```

### Checkbox List

Multiple checkboxes for option selection

```tsx
const [selectedOptions, setSelectedOptions] = useState({
    option1: false,
    option2: true,
    option3: false,
})

<div className="space-y-3">
    <Checkbox
        checked={selectedOptions.option1}
        onCheckedChange={(checked) =>
            setSelectedOptions(prev => ({ ...prev, option1: !!checked }))
        }
    >
        Enable dark mode
    </Checkbox>
    <Checkbox
        checked={selectedOptions.option2}
        onCheckedChange={(checked) =>
            setSelectedOptions(prev => ({ ...prev, option2: !!checked }))
        }
    >
        Show desktop notifications
    </Checkbox>
    <Checkbox
        checked={selectedOptions.option3}
        onCheckedChange={(checked) =>
            setSelectedOptions(prev => ({ ...prev, option3: !!checked }))
        }
    >
        Auto-save documents
    </Checkbox>
</div>
```

### Form Integration Example

Checkbox integrated with form handling

```tsx
interface FormData {
    marketing: boolean
    updates: boolean
    terms: boolean
}

const [formData, setFormData] = useState<FormData>({
    marketing: false,
    updates: true,
    terms: false,
})

const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault()
    if (!formData.terms) {
        // Handle validation error
        return
    }
    console.log('Form submitted:', formData)
}

;<form onSubmit={handleSubmit} className="space-y-4">
    <Checkbox
        checked={formData.marketing}
        onCheckedChange={(checked) =>
            setFormData((prev) => ({ ...prev, marketing: !!checked }))
        }
        subtext="Receive promotional emails and special offers"
    >
        Marketing communications
    </Checkbox>

    <Checkbox
        checked={formData.updates}
        onCheckedChange={(checked) =>
            setFormData((prev) => ({ ...prev, updates: !!checked }))
        }
        subtext="Important product updates and security notifications"
    >
        Product updates
    </Checkbox>

    <Checkbox
        checked={formData.terms}
        onCheckedChange={(checked) =>
            setFormData((prev) => ({ ...prev, terms: !!checked }))
        }
        required={true}
        error={!formData.terms}
    >
        I agree to the terms of service and privacy policy
    </Checkbox>

    <button type="submit">Submit</button>
</form>
```

## Behavior

### State Management

- **Controlled**: Use `checked` and `onCheckedChange` props for external state control
- **Uncontrolled**: Use `defaultChecked` for internal state management
- **Indeterminate**: Special state for partial selection scenarios

### Interactive States

- **Default**: Normal interactive state with hover effects
- **Hover**: Visual feedback on mouse hover
- **Disabled**: Non-interactive state with reduced opacity
- **Error**: Red styling to indicate validation errors

### Form Integration

- **Required Field**: Visual indicators and validation support
- **Value Handling**: Proper value attribute for form submissions
- **Validation**: Error state styling for failed validation

### Accessibility Features

- **Keyboard Navigation**: Tab navigation and Space key activation
- **Screen Reader**: Proper labeling and state announcements
- **Focus Indicators**: Clear visual focus indicators

## Best Practices

### When to Use

- **Multiple Selection**: When users need to select multiple options
- **Terms Acceptance**: For accepting terms, conditions, or agreements
- **Settings/Preferences**: For enabling/disabling features or options
- **Form Controls**: As part of larger forms for data collection

### Label Guidelines

- **Clear and Concise**: Use descriptive labels that clearly explain what will happen
- **Action-Oriented**: Frame labels as actions (e.g., "Enable notifications")
- **Positive Language**: Use positive phrasing when possible
- **Context Specific**: Provide additional context through subtext when needed

### Indeterminate State

- **Partial Selection**: Use when representing a partially selected state
- **Parent-Child Relationships**: Common in hierarchical lists or tree structures
- **Group Selection**: When some but not all items in a group are selected
