## Usage

```tsx
import { Slider, SliderVariant, SliderSize, SliderValueType } from 'blend-v1'
import { useState } from 'react'

function MyComponent() {
    const [value, setValue] = useState([25])

    return (
        <div>
            <label className="block text-sm font-medium mb-2">
                Volume: {value[0]}%
            </label>
            <Slider
                variant={SliderVariant.PRIMARY}
                size={SliderSize.MEDIUM}
                value={value}
                onValueChange={setValue}
                min={0}
                max={100}
                step={1}
                showValueLabels={true}
                labelPosition="top"
                valueFormat={{
                    type: SliderValueType.NUMBER,
                    suffix: '%',
                }}
            />
        </div>
    )
}
```

## Features

- **Range Selection**: Support for single and multi-thumb configurations for range selection
- **Value Formatting**: Flexible value formatting with numbers, percentages, decimals, and custom formatters
- **Visual Variants**: Primary and secondary styling variants for different contexts
- **Size Options**: Three size variants (small, medium, large) for different design needs
- **Value Labels**: Optional value labels with configurable positioning (top, bottom, inline)
- **Accessibility First**: Full keyboard navigation, ARIA compliance, and screen reader support
- **Touch Support**: Optimized for touch interactions on mobile devices
- **Built on Radix UI**: Leverages Radix UI Slider for robust functionality and accessibility
- **Custom Styling**: Comprehensive token system for complete visual customization
- **Orientation Support**: Horizontal and vertical slider orientations
- **Precise Control**: Configurable step increments and min/max ranges
- **Responsive Design**: Adapts to different screen sizes and input methods


### Range Slider

Multi-thumb slider for range selection

```tsx

const [priceRange, setPriceRange] = useState([200, 800])

        variant={SliderVariant.SECONDARY}

        max={1000}
        step={10}
        showValueLabels={true}

        valueFormat={{
            type: SliderValueType.NUMBER,
            prefix: '$'
        }}
    />
</div>
```

## Behavior

### Value Management

- **Controlled Mode**: Use `value` and `onValueChange` props for external state control
- **Uncontrolled Mode**: Use `defaultValue` for component-managed state
- **Multi-thumb**: Array values create multiple thumbs for range selection
- **Value Constraints**: Values are automatically constrained to min/max bounds

### Interaction Patterns

- **Thumb Dragging**: Click and drag thumbs to adjust values
- **Track Clicking**: Click on track to jump thumb to that position
- **Keyboard Navigation**: Arrow keys for precise value adjustment
- **Touch Support**: Optimized touch interactions for mobile devices

### Formatting System

- **Type-based**: Different formatting based on SliderValueType
- **Custom Formatters**: Use custom functions for complex formatting
- **Prefix/Suffix**: Add text before or after values
- **Precision Control**: Configurable decimal places for display

### Accessibility Features

- **Keyboard Navigation**: Full keyboard support with arrow keys, Page Up/Down, Home/End
- **ARIA Labels**: Proper ARIA roles and labels for screen readers
- **Focus Management**: Clear focus indicators and logical tab order
- **Value Announcements**: Screen reader announcements for value changes

## Best Practices

### When to Use Sliders

- **Range Selection**: For selecting values within a continuous range
- **Settings Adjustment**: For adjusting settings like volume, brightness, or opacity
- **Filtering**: For range-based filtering in search or data views
- **Quantity Selection**: For selecting quantities with clear min/max bounds

### Slider Design Guidelines

- **Clear Labels**: Always provide clear labels indicating what the slider controls
- **Appropriate Ranges**: Choose meaningful min/max values and step increments
- **Value Feedback**: Show current values either in labels or through formatting
- **Consistent Sizing**: Use consistent slider sizes throughout your application

### Value Formatting

- **Meaningful Units**: Use appropriate units (%, $, Â°C) that users understand
- **Precision Levels**: Match decimal precision to the use case (whole numbers vs decimals)
- **Custom Formatters**: Use custom formatters for complex business logic
- **Visual Feedback**: Consider showing formatted values in real-time

### Accessibility Considerations

- **Keyboard Support**: Ensure all functionality is available via keyboard
- **Screen Readers**: Test with screen readers to verify announcements
- **Color Independence**: Don't rely solely on color to convey state
- **Touch Targets**: Ensure thumb areas are large enough for touch interaction

### Mobile Considerations

- **Touch Targets**: Ensure thumbs are large enough for accurate touch interaction
- **Gesture Support**: Test drag gestures on various mobile devices
- **Orientation**: Consider how sliders work in both portrait and landscape modes
- **Performance**: Optimize for smooth animations on lower-powered devices
