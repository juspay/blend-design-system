<ChangelogCard summary="Charts Component" prId={["346", "330"]} commitHash={["79a9893", "d185ad0"]}>

<ChangelogEntry type="feat" component="Charts">
    **Empty State Handling** - Added comprehensive empty state handling for
    charts, providing meaningful feedback when no data is available to display.
</ChangelogEntry>

<ChangelogEntry type="fix" component="Charts">
    **Typography Improvements** - Fixed font size inconsistencies for x/y axis
    labels, legends, and chart text elements for better readability and design
    consistency.
</ChangelogEntry>

<ChangelogEntry type="fix" component="Charts">
    **Axis Spacing** - Resolved inconsistent spacing between consecutive ticks
    on the x-axis, ensuring uniform visual spacing and improved chart
    readability.
</ChangelogEntry>

<ChangelogEntry type="fix" component="Charts">
    **Legend Consistency** - Fixed "+n more" legend inconsistency issues,
    providing reliable legend display for charts with multiple data series.
</ChangelogEntry>

<ChangelogEntry type="fix" component="Charts">
    **Tick Overlap Prevention** - Handled overlapping of first x/y axis ticks,
    ensuring all tick labels remain visible and readable regardless of data
    density.
</ChangelogEntry>

<ChangelogEntry type="feat" component="Charts">
    **ChartContainer Component** - Introduced reusable ChartContainer wrapper
    with consistent styling including borders, background, and radius for
    unified chart presentation.
</ChangelogEntry>

<ChangelogEntry type="feat" component="Charts">
    **CoreChart Component** - Introduced CoreChart for pure rendering with
    external state support for hover and selection interactions, enabling
    advanced chart customization.
</ChangelogEntry>

<ChangelogEntry type="feat" component="Charts">
    **Advanced Date/Time Formatting** - Added comprehensive date/time formatting
    system with timezone support (IANA zones like UTC, EST, PST), 12/24-hour
    format via `hour12` prop, smart formatting that adapts to time context, and
    support for multiple input formats (POSIX, UTC strings, JS timestamps).
</ChangelogEntry>

</ChangelogCard>
