## Usage

```tsx
import { ButtonGroup, Button, ButtonType, ButtonSize } from 'blend-v1'

function MyComponent() {
    return (
        <ButtonGroup stacked={false}>
            <Button
                text="Cancel"
                buttonType={ButtonType.SECONDARY}
                size={ButtonSize.MEDIUM}
                onClick={() => console.log('Cancel clicked')}
            />
            <Button
                text="Save Changes"
                buttonType={ButtonType.PRIMARY}
                size={ButtonSize.MEDIUM}
                onClick={() => console.log('Save clicked')}
            />
        </ButtonGroup>
    )
}
```

## Features

- **2 Layout Modes**: Horizontal (default) and vertical stacked layouts for different design contexts
- **Automatic Positioning**: Intelligent button positioning with left, center, and right positioning in stacked mode
- **Seamless Integration**: Works exclusively with Button components for consistent styling
- **Smart Spacing**: Automatic gap management - 10px for horizontal, 0px for stacked layout
- **Dynamic Children**: Supports any number of Button components as children
- **Position Awareness**: Automatically sets `buttonGroupPosition` prop for stacked buttons
- **Flexible Styling**: Inherits all Button component styling capabilities
- **Responsive Design**: Adapts to container constraints and screen sizes
- **Type Safety**: TypeScript support with proper Button component validation
- **Clean Rendering**: Optimized rendering with React.cloneElement for stacked mode

## Component Tokens

ButtonGroup uses the Button component's tokens for styling. The container itself has minimal styling:

```tsx
export type ButtonGroupTokenType = {
    spacing: {
        horizontal: CSSObject['gap'] // 10px gap between buttons
        stacked: CSSObject['gap'] // 0px gap for connected buttons
    }
    layout: {
        horizontal: {
            display: CSSObject['display'] // 'flex'
            flexDirection: CSSObject['flexDirection'] // 'row'
        }
        stacked: {
            display: CSSObject['display'] // 'flex'
            flexDirection: CSSObject['flexDirection'] // 'column'
        }
    }
}

// Individual buttons inherit these positioning props in stacked mode:
export type ButtonGroupPosition = 'left' | 'center' | 'right'
```

## Behavior

### Layout Modes

- **Horizontal (Default)**: Buttons arranged side-by-side with 10px gap
- **Stacked**: Buttons arranged vertically with 0px gap for connected appearance

### Automatic Positioning

- **First Button**: Gets `buttonGroupPosition="left"` in stacked mode
- **Middle Buttons**: Get `buttonGroupPosition="center"` in stacked mode
- **Last Button**: Gets `buttonGroupPosition="right"` in stacked mode
- **Single Button**: Gets `buttonGroupPosition="left"` in stacked mode

### Children Management

- **Type Safety**: Only accepts Button components as children
- **Dynamic Rendering**: Automatically handles single or multiple Button children
- **Prop Injection**: Adds positioning props to buttons in stacked mode
- **Key Management**: Preserves existing keys or generates index-based keys

### Responsive Behavior

- **Container Adaptation**: Adapts to parent container constraints
- **Button Scaling**: Individual buttons maintain their responsive behavior
- **Overflow Handling**: Horizontal groups may wrap based on Button properties

## Accessibility

- **Focus Management**: Maintains proper tab order through button sequence
- **Keyboard Navigation**: Standard button keyboard navigation (Tab, Enter, Space)
- **Screen Reader Support**: Each button maintains its individual accessibility features
- **Semantic Structure**: Uses proper button semantics for each child component
- **ARIA Compliance**: Inherits ARIA attributes from individual Button components
- **High Contrast**: Supports high contrast mode through Button component styling

## Best Practices

### When to Use Horizontal Layout

- Form action buttons (Cancel, Submit)
- Dialog confirmations (No, Yes)
- Navigation controls (Previous, Next)
- Toolbar actions with limited buttons

### When to Use Stacked Layout

- Vertical navigation menus
- Action lists in sidebars
- Mobile-first button groups
- When vertical space is preferred over horizontal

### Button Combinations

- **Primary + Secondary**: Most common pattern for forms
- **Multiple Secondary**: For equal-weight actions
- **Danger + Secondary**: For destructive confirmations
- **Success + Secondary**: For positive confirmations
